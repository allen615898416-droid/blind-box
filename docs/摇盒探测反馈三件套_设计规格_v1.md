# 摇盒探测反馈三件套 · 设计规格

> 版本 v1.0 · 2026-06-15 · GameDesigner
> 用途：定义「声音 + 震动 + 动画幅度」探测反馈系统的完整规格，可直接进开发。
> 素材基础：3 段真实盲盒摇晃录音（`摇晃1_clean.wav/mp3`、`摇晃2_clean.wav/mp3`、`摇晃3_clean.wav/mp3`），已做降噪+静音切除+压缩+响度标准化。
> 上层文档：`盲盒摇摇乐_重构骨架_GDD_v2.md` · `盲盒摇摇乐_策划需求文档_v3.md` · `猜盲盒_核心页面与机制设计稿_v1.md`

---

## 一、设计原则（来自上层 GDD 的约束）

| 原则 | 说明 | 本规格如何遵守 |
|------|------|--------------|
| **主观感受，不量化** | 不放波形/数值/关键词读数，要玩家凭直觉 | 三件套都是"现象级"输出，无任何文字标签 |
| **动画幅度是核心通道** | 不戴耳机时靠视觉差异玩 | 动画幅度参数独立可调，与声音解耦 |
| **甜区控制** | 主线差别大、挑战关差别小 | `spread` 旋钮统一控制三件套的差异度 |
| **清晰度噪声** | 每次探测加微量随机扰动 | `clarity` 旋钮控制每次摇的随机抖动幅度 |
| **满盒摇不动** | 填充度高的盒→极弱反馈 | 音量/震动强度/动画幅度都受 `fill` 压制 |
| **配件声可辨** | 有独立配件 → 清脆"咔哒"层 | 用不同源音效 + EQ 增强 |

## 二、音效基因参数系统（核心数据模型）

> **核心思路**：3 段真实录音是"素材基因库"，每个盒子类型选一段做底，通过参数变换产生差异。**不是给5种盒配5段录音，而是3段录音×参数变换 = 无限组合**。

### 2.1 音效基因结构

```
ShakeGene = {
  source:     1 | 2 | 3,    // 使用哪段 clean 音效做底
  pitch:      -6 ~ +6,      // 半音偏移（负=沉闷重物，正=轻快小件）
  speed:      0.6 ~ 1.4,    // 播放速率（<1 慢放=重/满，>1 快放=轻/空）
  eqLowGain:  -12 ~ +6,     // 低频增益 dB（+低频=空洞共鸣，-低频=闷实）
  eqHighGain: -12 ~ +6,     // 高频增益 dB（+高频=清脆配件感，-高频=蒙住）
  reverbMix:  0 ~ 0.5,      // 混响干湿比（0=紧实，0.5=空洞回响）
  volume:     0.1 ~ 1.0,    // 基础音量（满盒极低，空盒高）
}
```

### 2.2 五种盒型的基因配方（`[待测]`，需 playtest 确认）

| 盒型 | source | pitch | speed | eqLowGain | eqHighGain | reverbMix | volume | 设计意图 |
|------|--------|-------|-------|-----------|------------|-----------|--------|---------|
| **满盒大款** | 1 | -4 | 0.7 | -8 | -6 | 0 | 0.2 | 沉、闷、短——"摇不动" |
| **配件款** | 3 | +2 | 1.0 | -2 | +4 | 0.1 | 0.7 | 中等+清脆高频层——"咔哒咔哒" |
| **空盒轻量款** | 2 | +4 | 1.3 | +4 | +2 | 0.35 | 0.9 | 轻、空、回响——"六面晃" |
| **翅膀配件款** | 1 | -1 | 0.85 | 0 | +3 | 0.08 | 0.55 | 偏重但有脆点——"头重偶尔叮" |
| **底座独立款** | 3 | -3 | 0.8 | +2 | -3 | 0.2 | 0.6 | 偏空+闷+回响——"闷盒砸底" |

### 2.3 素材-盒型匹配理由

| 素材 | 特征 | 适配盒型 |
|------|------|---------|
| **摇晃1_clean** (1.2s, ~6次密集冲击) | 短促、冲击密集、衰减快 | 满盒（密集闷响）+ 翅膀款（偶尔脆点） |
| **摇晃2_clean** (2.0s, ~10+散碎撞击) | 持续长、撞击散碎、空间大 | 空盒（六面晃） |
| **摇晃3_clean** (1.4s, ~8次渐进冲击) | 中等时长、有递进、尾音稍长 | 配件款（双层感）+ 底座款（闷+回响） |

## 三、震动映射规则

> **原则**：震动是声音的触觉投影，不做独立设计。从音效基因推导。

### 3.1 映射公式

```
震动模式 = [pause_ms, buzz_ms, pause_ms, buzz_ms, ...]

其中：
  播放节奏 → 震动节奏：  speed 越大，pause 越短（快摇=密震）
  pitch    → 震动频率：  pitch < 0 → 长 buzz (40-65ms，沉)
                          pitch > 0 → 短 buzz (15-30ms，脆)
  volume   → 震动强度：  iOS 不支持强度，用 buzz 时长近似
  reverbMix → 尾震：     reverbMix > 0.2 → 末尾追加 1-2 次弱震（模拟回响）
```

### 3.2 五种盒型的震动模式特征

| 盒型 | 节奏 | 单次时长 | 次数 | 尾震 | 体感描述 |
|------|------|---------|------|------|---------|
| 满盒大款 | 慢(80-120ms间隔) | 长(50ms) | 1-2 | 无 | "咚——咚"，沉闷 |
| 配件款 | 中(50-80ms) | 短(20ms) + 夹杂极短(12ms) | 4-6 | 无 | "哒哒咔哒"，双层 |
| 空盒轻量款 | 快(30-50ms) | 短(18ms) | 6-10 | 有1次 | "哒哒哒哒—哒"，碎+回 |
| 翅膀配件款 | 中慢(60-90ms) | 中(30ms)+偶尔极短(12ms) | 3-5 | 无 | "咚…咔…咚"，头重脚轻 |
| 底座独立款 | 慢(70-100ms) | 中长(40ms) | 3-4 | 有1-2次 | "咚…咚……咚"，闷+尾 |

### 3.3 实现规范

- **Android**: `navigator.vibrate(pattern)` 直接传 pattern 数组
- **iOS**: 浏览器不支持震动，用**画面脉冲补偿**（盒子周围扩散圈 + 缩放弹跳，见五·3）
- **Web Demo**: 同 Android 逻辑，无震动时同 iOS 补偿

## 四、动画幅度映射规则

> **原则**：动画幅度是**不戴耳机时的主线索通道**，必须有可分辨差异。不是装饰。

### 4.1 映射公式

```
ShakeAnimation = {
  amplitudeX:  (1 - fill) × baseAmp × (1 + looseness×0.5)    // 水平晃幅(px)
  amplitudeY:  fill < 0.4 ? (1-fill)×baseAmp×0.7 : 2         // 垂直晃幅(空盒大幅上下,满盒微颤)
  rotation:    (1 - fill) × baseRot × (1 + looseness×0.3)    // 旋转角度(deg)
  speed:       baseSpeed × (0.5 + (1-fill)×0.5 + looseness×0.2)  // 动画周期(s)
  bounce:      reverbMix × 0.4                                  // 回弹余震(0~0.2)
}

baseAmp = 8px, baseRot = 5deg, baseSpeed = 0.5s

fill 和 looseness 从音效基因推导：
  fill   ≈ 1 - volume × (0.5 + speed×0.3)        // 音量大+快 = 空
  looseness ≈ reverbMix × 0.7 + (eqLowGain/12)×0.3  // 回响大+低频多 = 松
```

### 4.2 五种盒型的动画特征

| 盒型 | X晃幅 | Y晃幅 | 旋转 | 周期 | 余震 | 视觉体感 |
|------|-------|-------|------|------|------|---------|
| 满盒大款 | 2px | 1px | 1° | 0.6s | 无 | 微颤，"摇不动" |
| 配件款 | 6px | 4px | 3° | 0.45s | 无 | 中等晃，偶有脆跳 |
| 空盒轻量款 | 12px | 8px | 7° | 0.3s | 有 | 大幅快晃，"六面乱飞" |
| 翅膀配件款 | 5px | 3px | 4° | 0.5s | 无 | 中等偏沉，偶尔抖 |
| 底座独立款 | 8px | 5px | 3° | 0.55s | 有 | 闷沉地晃，有尾 |

### 4.3 关键帧动画规范

```css
@keyframes shake-box {
  0%, 100% { transform: translate(0, 0) rotate(0); }
  15%      { transform: translate(calc(var(--ampX) * -1), calc(var(--ampY) * -0.7)) rotate(calc(var(--rot) * -1)); }
  30%      { transform: translate(var(--ampX), calc(var(--ampY) * 0.3)) rotate(var(--rot)); }
  45%      { transform: translate(calc(var(--ampX) * -0.6), calc(var(--ampY) * -0.4)) rotate(calc(var(--rot) * -0.5)); }
  60%      { transform: translate(calc(var(--ampX) * 0.4), 0) rotate(calc(var(--rot) * 0.3)); }
  80%      { transform: translate(calc(var(--ampX) * -0.2), 0) rotate(calc(var(--rot) * -0.15)); }
}
```

- 动画通过 CSS 变量 `--ampX`, `--ampY`, `--rot` 驱动，JS 只设变量值
- `animation-timing-function: cubic-bezier(.36,.07,.19,.97)` 模拟物理惯性
- 余震：主动画结束后追加 1-2 周期极弱（amplitude×0.15）动画

## 五、甜区调控机制

### 5.1 差异度 (`spread`: 0~1)

**作用**：控制五种盒型之间的基因差异。`spread=1` 是配方原始值（最大差异），`spread=0` 全部收敛到中心点（无法区分）。

**实现**：每条基因参数向中心值线性插值：
```
actual = center + (gene - center) × spread
```

| 难度 | spread | 体验 |
|------|--------|------|
| 主线 | 0.80-0.95 | 一摇就感觉到不同 |
| 挑战 | 0.30-0.45 | 要反复对比才分辨 |

### 5.2 清晰度 (`clarity`: 0~1)

**作用**：每次探测时加随机扰动。`clarity=1` 无扰动（每次摇一模一样），`clarity=0` 大幅扰动（同一盒两次摇感觉不同）。

**实现**：每次播放时对基因参数加高斯噪声：
```
noise_amplitude = (1 - clarity) × 0.3
actual_pitch  = gene.pitch  + randn() × noise_amplitude × 3
actual_speed  = gene.speed  + randn() × noise_amplitude × 0.15
actual_volume = gene.volume + randn() × noise_amplitude × 0.15
```

- 主线 `clarity = 0.85-0.90`
- 挑战 `clarity = 0.40-0.55`

### 5.3 甜区验证方法

| 测试 | 目的 | 通过标准 |
|------|------|---------|
| **A/B 对比测试** | 两个盒摇一摇，能不能说出谁更重/更空 | 主线>90%正确率，挑战50-70% |
| **三局上手测试** | 新手前3关是否不用教就自然开始对比 | 不看任何说明就能开始玩 |
| **去耳机测试** | 不戴耳机仅凭动画幅度玩主线 | 正确率>80% |
| **重复可辨测试** | 同一盒摇3次，感受是否一致 | 主线一致感>85%，挑战有微妙变化但可辨 |

| `摇晃1_clean.wav` | 44.1kHz / 16bit / Mono | 1.2s | 满盒/翅膀款底音 |
| `摇晃2_clean.wav` | 44.1kHz / 16bit / Mono | 2.0s | 空盒底音 |
| `摇晃3_clean.wav` | 44.1kHz / 16bit / Mono | 1.4s | 配件/底座款底音 |
| `摇晃1_single.wav` | 同上 | 0.3s | 单次冲击音效（UI反馈等） |
| `摇晃2_single.wav` | 同上 | 0.1s | 单次脆响（揭晓动画） |
| `摇晃3_single.wav` | 同上 | 0.1s | 单次闷响（揭晓动画） |
| `shake_combined.wav` | 同上 | 4.55s | 连摇演示（非游戏内用） |

### 6.2 游戏内加载策略

```
优先加载 MP3 版本（体积小）：
  摇晃1_clean.mp3  8.5KB
  摇晃2_clean.mp3  15KB
  摇晃3_clean.mp3  10KB

总音效预算 < 50KB（满足九·资源规格上限"单音效<100KB"）
```

### 6.3 Web Audio 实时处理链

```
[AudioBufferSourceNode] → [playbackRate] → [GainNode(volume)]
  → [BiquadFilter(lowshelf, eqLowGain)]
  → [BiquadFilter(highshelf, eqHighGain)]
  → [ConvolverNode(reverbIR)] → [dry/wet mix]
  → [destination]

pitch 通过 playbackRate 实现: rate = 2^(semitones/12)
reverb 通过小型 IR (impulse response) 实现, 预生成 3 种尺寸
```

## 七、BGM 方案

> **核心原则**：摇盒/下注/揭晓都是瞬间动作（几秒），切 BGM 反而打断体验。BGM 只在"停留场景"循环，核心游戏页全程不切。

### 7.1 BGM 分配

| 场景 | BGM | 音量 | 说明 |
|------|-----|------|------|
| P1 首页 / P2 收集墙 | `collection_Fluffing_a_Duck.mp3` | 35% | 玩家浏览、选择，待最久，轻松氛围 |
| P3 核心游戏页（摇+探+注+揭晓） | `shake_explore_Itty_Bitty_8_Bit.mp3` | 18% | 全程不切，8-bit 风格贴合摇盒游戏感 |

**不使用的素材**（留素材库，以后活动主题/特殊关卡可用）：
- `confirm_bet_Sneaky_Snitch.mp3`
- `reveal_Monkeys_Spinning_Monkeys.mp3`

### 7.2 场景切换逻辑

```
进入 P1/P2 → fade in collection BGM
进入 P3   → crossfade → game BGM（全程循环，不中断）
  ├─ 摇盒瞬间 → SFX 音效盖住 BGM（音量远高于 BGM）
  ├─ 下注瞬间 → 不切 BGM
  └─ 揭晓瞬间 → SFX sting（3秒短音效）
回到 P1/P2 → crossfade → collection BGM
```

### 7.3 Crossfade 实现

- 切换时旧 BGM fade out（40ms 步进，约 0.5s）
- 新 BGM fade in（同速率）
- 同场景内不触发 crossfade（`_bgmScene === scene` 时跳过）

### 7.4 SFX 与 BGM 的关系

| 事件 | SFX 类型 | 与 BGM 关系 |
|------|---------|------------|
| 摇盒 | gene 驱动的 clean 音效 | SFX 音量远高于 BGM，自然盖住 |
| 揭晓 | single sting（~3s） | 短暂高光音效，盖住 BGM |
| 收集点亮 | 收集确认音效 | 叠加在 collection BGM 上 |

## 八、三件套时序同步规范

> **核心约束**：三件套必须**帧级同步**——声音的撞击点、震动的 buzz、动画的关键帧必须对齐。

### 8.1 同步机制

```
1. 播放音效（AudioBufferSourceNode.start(t0)）
2. 从 AudioBuffer 分析出撞击时间点（预计算，存在 ShakeGene.hitTimes[]）
3. 用 hitTimes[] 驱动：
   - 震动: 在 t0 + hitTimes[i] 时发 buzz
   - 动画: 主循环持续，但在 hitTimes[i] 时微调 amplitude（冲击瞬间放大 20%）
   - 视觉脉冲(iOS补偿): 在 hitTimes[i] 时触发扩散圈
```

### 8.2 hitTimes 预计算

```
对每段 clean 音效做 onset detection（能量包络峰值检测）:
  - 滑动窗口 RMS (20ms)
  - 峰值检测，阈值 = max_RMS × 0.3
  - 最小间距 50ms（合并过近的峰值）

结果（示例）:
  摇晃1_clean: [0.02, 0.15, 0.28, 0.42, 0.58, 0.72]  (6次)
  摇晃2_clean: [0.01, 0.08, 0.18, 0.28, 0.42, 0.55, 0.68, 0.82, 0.95, 1.15, 1.35, 1.52, 1.70, 1.85]  (14次)
  摇晃3_clean: [0.02, 0.12, 0.25, 0.38, 0.55, 0.70, 0.85, 1.02, 1.18]  (9次)
```

> `hitTimes` 在加载音效时一次性计算，缓存到基因数据中。

## 九、探测交互流程（单次摇盒的完整时序）

```
玩家点"摇一摇"
  │
  ├─ 0ms   按钮按下缩放反馈
  ├─ 50ms  ApplyGene(): 根据 gene + spread + clarity 计算实际参数
  ├─ 60ms  播放音效 (Web Audio start)
  ├─ 60ms  启动震动 pattern (navigator.vibrate)
  ├─ 60ms  启动盒子 CSS 动画 (set --ampX/Y/rot, add .shaking)
  │
  ├─ 60ms+hitTimes[i]  每个撞击点:
  │   ├─ 动画 amplitude 微调 (冲击瞬间 +20%)
  │   ├─ iOS 补偿脉冲
  │   └─ 方向指示灯闪烁 (若启用)
  │
  ├─ duration + 200ms  动画结束 (remove .shaking)
  ├─ duration + 300ms  余震动画 (若有)
  └─ duration + 500ms  状态归位, 可再次摇
```

## 十、与现有 Demo (`figure-shake-lab.html`) 的关系

| 维度 | 现有 Demo | 本规格 |
|------|----------|--------|
| 音源 | Web Audio 合成 (oscillator + noise) | **真实录音** + 实时参数变换 |
| 震动 | 基于合成撞击表生成 pattern | 基于真实录音的 onset detection 生成 pattern |
| 动画 | CSS 变量驱动（fill/looseness → amp/rot/speed） | 同架构，但参数从基因推导而非硬编码 |
| 差异度/清晰度 | 已有旋钮 | 保留，但底层从合成参数改为基因插值 |
| 盒型 | 5 种 (mass/fill/parts/looseness/cgX/cgZ) | 5 种 (source/pitch/speed/eqLow/eqHigh/reverb/volume) |

**迁移策略**：保留现有 Demo 的 UI 骨架和交互模式，替换音源为真实录音 + 基因系统。
