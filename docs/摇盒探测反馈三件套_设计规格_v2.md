# 摇盒探测反馈三件套 · 设计规格

> 版本 v2.0 · 2026-06-16 · Allen & AI
> 用途：定义「声音 + 震动 + 动画幅度」探测反馈系统的完整规格，可直接进开发。
> 素材基础：3 段真实盲盒摇晃录音（`摇晃1_clean.wav/mp3`、`摇晃2_clean.wav/mp3`、`摇晃3_clean.wav/mp3`），已做降噪+静音切除+压缩+响度标准化。
> 上层文档：`ShakeSecret_策划需求文档_v6.md`

---

## 一、设计原则（来自上层 GDD 的约束）

| 原则 | 说明 | 本规格如何遵守 |
|------|------|--------------|
| **主观感受，不量化** | 不放波形/数值/关键词读数，要玩家凭直觉 | 三件套都是"现象级"输出，无任何文字标签 |
| **动画幅度是核心通道** | 不戴耳机时靠视觉差异玩 | 动画幅度参数独立可调，与声音解耦 |
| **甜区控制** | 主线差别大、挑战关差别小 | `spread` 旋钮统一控制三件套的差异度 |
| **清晰度噪声** | 每次探测加微量随机扰动 | `clarity` 旋钮控制每次摇的随机抖动幅度 |
| **满盒摇不动** | 填充度高的盒→极弱反馈 | 音量/震动强度/动画幅度都受 `fill` 压制 |
| **配件声可辨** | 有独立配件 → 清脆"咔哒"层 | 用不同源音效 + EQ 增强 |
| **物理基因驱动** | 角色在盒中的物理状态决定一切 | 4 维物理基因 → 公式推导 → 7 维音频基因 → 三件套 |

## 二、物理基因 → 音频基因系统（核心数据模型）

> **v2 核心变化**：从 5 种硬编码盒型 → 8+1 角色物理基因系统。
> 每个角色定义 4 维物理参数（fill/rattle/damping/size），通过 `physToGene()` 公式自动推导 7 维音频参数。
> 新系列角色只需填 4 个物理参数，音色自动计算。

### 2.1 四维物理基因

```
PhysicsGene = {
  fill:    0 ~ 1,    // 盒内填充度（0=空盒 1=塞满摇不动）
  rattle:  0 ~ 1,    // 内容松散度（0=固定一体 1=散乱晃荡）
  damping: 0 ~ 1,    // 内衬阻尼（0=裸放撞击 1=厚包装缓冲）
  size:    0 ~ 1,    // 角色体型（0=迷你 1=大型）
}
```

| 基因 | 范围 | 物理含义 | 听感影响 |
|------|------|---------|---------|
| **fill** | 0~1 | 盒内填充度（0=空 1=塞满） | 音量↓ 晃幅↓ 沉闷感↑ |
| **rattle** | 0~1 | 内容松散度（0=固定 1=散乱） | 高频脆响↑ 撞击次数↑ |
| **damping** | 0~1 | 内衬阻尼（0=裸放 1=厚包装） | 高频↓ 回响↓ 整体闷 |
| **size** | 0~1 | 角色体型（0=迷你 1=大型） | 低频↑ 音调↓ 速度↓ |

### 2.2 物理基因 → 7 维音频基因公式（`physToGene`）

```
function physToGene(p):
  f = p.fill, r = p.rattle, d = p.damping, s = p.size

  source   = r > 0.5 ? 2 : (f < 0.3 ? 1 : 0)     // 音效源选择
  pitch    = round((1-s)*8 + r*3 - f*4 - 6)        // 半音偏移
  speed    = 0.6 + (1-f)*0.5 + r*0.3                // 播放速率
  eqLow    = round((1-f)*10 + s*2 - 6)              // 低频增益 dB
  eqHigh   = round(r*12 - d*10 - 2)                 // 高频增益 dB
  reverb   = max(0, (1-f)*(1-d)*0.5)                // 混响干湿比
  volume   = max(0.1, (1-f)*0.8 + r*0.2)            // 基础音量
```

**设计意图**：
- `fill` 高 → 音量低/速度慢/晃幅小 → "摇不动"
- `rattle` 高 → 高频强/音源3/撞击多 → "散乱咔哒"
- `damping` 高 → 高频弱/回响小 → "蒙住"
- `size` 大 → 音调低/速度慢/低频强 → "沉闷大件"

### 2.3 音效基因结构（公式推导结果）

```
ShakeGene = {
  source:  0 | 1 | 2,     // 使用哪段 clean 音效做底
  pitch:   -6 ~ +6,       // 半音偏移（负=沉闷重物，正=轻快小件）
  speed:   0.6 ~ 1.4,     // 播放速率（<1 慢放=重/满，>1 快放=轻/空）
  eqLow:   -6 ~ +6,      // 低频增益 dB
  eqHigh:  -12 ~ +10,     // 高频增益 dB
  reverb:  0 ~ 0.5,       // 混响干湿比
  volume:  0.1 ~ 1.0,     // 基础音量
}
```

### 2.6 基因中心点与 spread 插值

```
GENE_CENTER = { pitch: 0, speed: 1.0, eqLow: -2, eqHigh: 0, reverb: 0.12, volume: 0.55 }

// spread 插值：所有角色基因向中心点收缩
actual = GENE_CENTER[param] + (gene[param] - GENE_CENTER[param]) × spread
```

- `spread = 1.0`：角色间差异最大（原始值）
- `spread = 0.0`：所有角色收敛到中心点（无法区分）

## 三、震动映射规则

> **原则**：震动是声音的触觉投影，不做独立设计。从音效基因推导。

### 3.1 映射公式

```
震动模式 = [pause_ms, buzz_ms, pause_ms, buzz_ms, ...]

其中：
  source+speed → 震动节奏：  使用 hitTimes[source] 映射节奏，speed 越大间隔越短
  pitch        → 震动频率：  pitch < 0 → 长 buzz (40-65ms，沉)
                            pitch > 0 → 短 buzz (15-30ms，脆)
  volume       → 震动强度：  iOS 不支持强度，用 buzz 时长近似
  reverb       → 尾震：     reverb > 0.2 → 末尾追加 1-2 次弱震（模拟回响）
```

### 3.2 各角色的震动特征

| 角色 | 节奏 | 单次时长 | 次数 | 尾震 | 体感描述 |
|------|------|---------|------|------|---------|
| **Pip** | 快(30-50ms) | 短(18ms) | 6-10 | 有1次 | "哒哒哒哒—哒"，碎+回 |
| **Mochi** | 中(50-80ms) | 短(20ms) | 5-8 | 无 | "哒 哒 哒 哒"，温和 |
| **Biscuit** | 中(50-70ms) | 短(15ms) + 极短(12ms) | 4-6 | 无 | "哒咔 哒咔"，双层 |
| **Lumi** | 中(50-70ms) | 中(22ms)+偶尔极短(12ms) | 4-7 | 无 | "咚…咔…咚"，头重脚轻 |
| **Crème Puff** | 中慢(60-90ms) | 短(18ms) | 3-5 | 无 | "哒…哒…哒"，软弹 |
| **Daisy** | 中(55-80ms) | 短(20ms) | 4-6 | 无 | "哒 哒咔 哒"，偶脆 |
| **Hugo** | 慢(80-120ms) | 长(50ms) | 1-2 | 无 | "咚——咚"，沉闷 |
| **Rex** | 慢(70-100ms) | 中长(40ms) | 3-4 | 有1-2次 | "咚…咚……咚"，闷+尾 |
| **Mystery** ⭐ | 慢(70-90ms) | 中长(35ms) | 3-4 | 有1次 | "咚…咚…咚—嗡"，闷+回 |

### 3.3 实现规范

- **Android**: `navigator.vibrate(pattern)` 直接传 pattern 数组
- **iOS**: 浏览器不支持震动，用**画面脉冲补偿**（盒子周围扩散圈 + 缩放弹跳，见五·3）
- **Web Demo**: 同 Android 逻辑，无震动时同 iOS 补偿

## 四、动画幅度映射规则

> **原则**：动画幅度是**不戴耳机时的主线索通道**，必须有可分辨差异。不是装饰。

### 4.1 映射公式

```
从音频基因推导 fill / loose / space：

  fill   = max(0, min(1, 1 - volume × (0.5 + speed×0.3)))
  loose  = max(0, min(1, reverb×0.7 + (eqLow/12)×0.3))
  space  = 1 - fill

动画参数：
  --ax  = 2 + space×10 + loose×6              (水平晃幅 px)
  --ay  = fill < 0.4 ? (1-fill)×8×0.7 : 2    (垂直晃幅 px)
  --rot = 1 + space×6 + loose×3               (旋转 deg)
  speed = 0.6 - space×0.18 - loose×0.08       (周期 s)
```

### 4.2 各角色的动画特征

| 角色 | fill | loose | space | X晃幅 | Y晃幅 | 旋转 | 周期 | 视觉体感 |
|------|------|-------|-------|-------|-------|------|------|---------|
| **Pip** | 0.13 | 0.28 | 0.87 | 12.4px | 5.0px | 7.0° | 0.42s | 大幅快晃，"六面乱飞" |
| **Mochi** | 0.52 | 0.08 | 0.48 | 7.7px | 2px | 4.0° | 0.50s | 温和晃荡 |
| **Biscuit** | 0.88 | 0.04 | 0.12 | 3.5px | 2px | 1.7° | 0.57s | 微颤 |
| **Lumi** | 0.38 | 0.26 | 0.62 | 10.4px | 3.5px | 5.6° | 0.46s | 中等偏快，偶有脆跳 |
| **Crème Puff** | 0.38 | 0.12 | 0.62 | 9.1px | 3.5px | 5.0° | 0.48s | 中等晃荡，软弹 |
| **Daisy** | 0.44 | 0.10 | 0.56 | 8.6px | 2px | 4.4° | 0.49s | 温和偏沉 |
| **Hugo** | 0.88 | 0.04 | 0.12 | 3.5px | 2px | 1.7° | 0.57s | 微颤，"摇不动" |
| **Rex** | 0.68 | 0.16 | 0.32 | 6.2px | 2px | 3.3° | 0.53s | 闷沉地晃，有尾 |
| **Mystery** ⭐ | 0.55 | 0.16 | 0.45 | 8.0px | 2px | 4.1° | 0.51s | 中等闷沉+余震 |

### 4.3 关键帧动画规范

```css
@keyframes shk {
  0%, 100% { transform: translate(0, 0) rotate(0); }
  15%      { transform: translate(calc(var(--ax)*-1), calc(var(--ay)*-1)) rotate(calc(var(--rot)*-1)); }
  30%      { transform: translate(var(--ax), calc(var(--ay)*.3)) rotate(var(--rot)); }
  45%      { transform: translate(calc(var(--ax)*-.6), calc(var(--ay)*-.4)) rotate(calc(var(--rot)*-.5)); }
  60%      { transform: translate(calc(var(--ax)*.4), 0) rotate(calc(var(--rot)*.3)); }
  80%      { transform: translate(calc(var(--ax)*-.2), 0) rotate(calc(var(--rot)*-.15)); }
}
```

- 动画通过 CSS 变量 `--ax`, `--ay`, `--rot` 驱动，JS 只设变量值
- `animation-timing-function: cubic-bezier(.36,.07,.19,.97)` 模拟物理惯性
- 余震：主动画结束后追加 1-2 周期极弱（amplitude×0.15）动画

## 五、甜区调控机制

### 5.1 差异度 (`spread`: 0~1)

**作用**：控制角色之间的基因差异。`spread=1` 是物理基因推导的原始值（最大差异），`spread=0` 全部收敛到中心点（无法区分）。

**实现**：每条基因参数向中心值线性插值：
```
actual = GENE_CENTER[param] + (gene[param] - GENE_CENTER[param]) × spread
```

| 难度 | spread | 体验 |
|------|--------|------|
| 主线 | 0.80-0.95 | 一摇就感觉到不同 |
| 挑战 | 0.30-0.45 | 要反复对比才分辨 |

### 5.2 清晰度 (`clarity`: 0~1)

**作用**：每次探测时加随机扰动。`clarity=1` 无扰动（每次摇一模一样），`clarity=0` 大幅扰动（同一盒两次摇感觉不同）。

**实现**：每次播放时对基因参数加高斯噪声：
```
noise_amplitude = (1 - clarity) × 0.3
actual_pitch  = gene.pitch  + randn() × noise_amplitude × 3
actual_speed  = gene.speed  + randn() × noise_amplitude × 0.15
actual_volume = gene.volume + randn() × noise_amplitude × 0.15
actual_eqLow  = gene.eqLow  + randn() × noise_amplitude × 4
actual_eqHigh = gene.eqHigh + randn() × noise_amplitude × 4
actual_reverb = gene.reverb + randn() × noise_amplitude × 0.1
```

- 主线 `clarity = 0.85-0.90`
- 挑战 `clarity = 0.40-0.55`

### 5.3 甜区验证方法

| 测试 | 目的 | 通过标准 |
|------|------|---------|
| **A/B 对比测试** | 两个盒摇一摇，能不能说出谁更重/更空 | 主线>90%正确率，挑战50-70% |
| **三局上手测试** | 新手前3关是否不用教就自然开始对比 | 不看任何说明就能开始玩 |
| **去耳机测试** | 不戴耳机仅凭动画幅度玩主线 | 正确率>80% |
| **重复可辨测试** | 同一盒摇3次，感受是否一致 | 主线一致感>85%，挑战有微妙变化但可辨 |

## 七、BGM 方案

> **核心原则**：摇盒/揭晓都是瞬间动作（几秒），切 BGM 反而打断体验。BGM 只在"停留场景"循环，核心游戏页全程不切。

### 7.1 BGM 分配

| 场景 | BGM | 音量 | 说明 |
|------|-----|------|------|
| P1 首页 / P2 收藏墙 | `collection_Fluffing_a_Duck.mp3` | 35% | 玩家浏览、选择，待最久，轻松氛围 |
| P3 核心游戏页（摇+揭晓） | `shake_explore_Itty_Bitty_8_Bit.mp3` | 18% | 全程不切，8-bit 风格贴合摇盒游戏感 |

### 7.2 场景切换逻辑

```
进入 P1/P2 → fade in collection BGM
进入 P3   → crossfade → game BGM（全程循环，不中断）
  ├─ 摇盒瞬间 → SFX 音效盖住 BGM（音量远高于 BGM）
  └─ 揭晓瞬间 → SFX sting（3秒短音效）
回到 P1/P2 → crossfade → collection BGM
```

### 7.3 Crossfade 实现

- 切换时旧 BGM fade out（40ms 步进，约 0.5s）
- 新 BGM fade in（同速率）
- 同场景内不触发 crossfade（`_bgmScene === scene` 时跳过）

### 7.4 SFX 与 BGM 的关系

| 事件 | SFX 类型 | 与 BGM 关系 |
|------|---------|------------|
| 摇盒 | gene 驱动的 clean 音效 | SFX 音量远高于 BGM，自然盖住 |
| 揭晓 | single sting（~3s） | 短暂高光音效，盖住 BGM |
| 收集点亮 | 收集确认音效 | 叠加在 collection BGM 上 |

## 八、三件套时序同步规范

> **核心约束**：三件套必须**帧级同步**——声音的撞击点、震动的 buzz、动画的关键帧必须对齐。

### 8.1 同步机制

```
1. 播放音效（AudioBufferSourceNode.start(t0)）
2. 从 AudioBuffer 分析出撞击时间点（预计算，存在 hitTimes[source]）
3. 用 hitTimes[source] 驱动：
   - 震动: 在 t0 + hitTimes[i] 时发 buzz
   - 动画: 主循环持续，但在 hitTimes[i] 时微调 amplitude（冲击瞬间放大 20%）
   - 视觉脉冲(iOS补偿): 在 hitTimes[i] 时触发扩散圈
```

### 8.2 hitTimes 预计算

```
对每段 clean 音效做 onset detection（能量包络峰值检测）:
  - 滑动窗口 RMS (20ms)
  - 峰值检测，阈值 = max_RMS × 0.25
  - 最小间距 50ms（合并过近的峰值）

结果（示例）:
  摇晃1_clean (source=0): [0.02, 0.15, 0.28, 0.42, 0.58, 0.72]  (6次)
  摇晃2_clean (source=1): [0.01, 0.08, 0.18, ..., 1.85]  (14次)
  摇晃3_clean (source=2): [0.02, 0.12, 0.25, ..., 1.18]  (9次)
```

> `hitTimes` 在加载音效时一次性计算，缓存到数组中，按 `source` 索引。

## 九、探测交互流程（单次摇盒的完整时序）

```
玩家点 "SHAKE"
  │
  ├─ 0ms   按钮按下缩放反馈
  ├─ 50ms  applySpread(gene) + applyClarity(gene): 计算实际参数
  ├─ 60ms  播放音效 (Web Audio start)
  ├─ 60ms  启动震动 pattern (navigator.vibrate)
  ├─ 60ms  启动盒子 CSS 动画 (set --ax/--ay/--rot, add .shaking)
  │
  ├─ 60ms+hitTimes[source][i]  每个撞击点:
  │   ├─ 动画 amplitude 微调 (冲击瞬间 +20%)
  │   └─ iOS 补偿脉冲
  │
  ├─ duration + 200ms  动画结束 (remove .shaking)
  └─ duration + 500ms  状态归位, 可再次摇
```
