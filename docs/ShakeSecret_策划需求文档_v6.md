# 摇盲盒（Shake Secret）· 策划需求文档

> 版本 v1.0 · 2026-06-16 · Allen & AI
> 用途：定义「摇盒猜角色」休闲游戏的完整产品需求，可直接进开发。

---

## 一、项目概览

| 项目项 | 内容 |
|--------|------|
| **项目名称** | Shake Secret（摇盲盒） |
| **品类** | 猜测/收集类休闲游戏 |
| **一句话介绍** | 摇盲盒，听声音、看晃幅、感受震动，猜里面藏着谁 |
| **目标平台** | 阶段一：Web/H5 Demo（验证核心循环）；阶段二：独立手游 App（iOS + Android）|
| **屏幕方向** | 竖屏（单手、碎片化） |
| **目标市场** | 首发东南亚，第二批欧美。语言：英文 + 泰文起步 |
| **目标用户** | 海外盲盒收集玩家、休闲游戏爱好者、碎片化时间消遣人群 |
| **变现模式** | 基础 IAA（广告变现），无付费墙、无内购货币（本阶段） |
| **竞品参考** | ① **Pokémon TCG Pocket**（收集+图鉴+晒卡，年收 $6.85 亿，证明收集驱动可上十亿）— 学其收集内核<br>② **Pop Mart / Pop Now 线下**（摇盒开箱仪式感、隐藏款）— 学其情感外壳与真实开盲盒体验<br>⚠️ ASMR Mystery Blind Bag 等"摇盒听声"类月收入<$100，作为**反面教材**：摇盒只能当外壳，不能当判定核心 |

**为什么值得做**：
- **好奇心驱动**：看到盒子就想知道里面是什么——这是最原始的人性。每组盲盒登场，第一秒就勾起"这里面是哪个"的好奇
- **小赌怡情**：信息不全时下注、选一个盒子、揭晓。"赌"是核心情绪，但失败近乎无痛——开出来不是目标也获得角色
- **真实体验线上化**："打盒听声"是盲盒圈真实玩家行为，有天然游戏化基础，但没有任何产品把这个物理体验搬到线上
- 物理基因系统提供可区分的听觉+触觉+视觉差异，不是纯随机猜，玩家的直觉有回报
- 盲盒IP天然适合系列扩展，每出一套新系列就是一波新内容

## 三、玩家体验目标

| 维度 | 描述 |
|------|------|
| **核心体验** | 探索与推理——"这盒里面到底是谁？"的好奇心驱动，加上揭晓瞬间的惊喜感 |
| **单局感受** | 短平快（30秒~2分钟）：选目标→摇→感受→猜→揭晓。摇盒时有紧张感（"听出来了吗？"），揭晓时有爽感或遗憾 |
| **长线感受** | 收集欲驱动——看收藏墙逐渐填满，追隐藏款，"全系列收集"的成就感 |
| **差异化卖点** | 不靠数值、不靠概率公示，靠**物理直觉**（声音、震动、晃动幅度）让玩家自己判断，模拟真实盲盒"打盒"体验 |

### 最爽的瞬间

- 摇了两三下就凭直觉锁定目标，Pick 一开果然中了
- 以为是个普通款，结果蹦出隐藏款 Mystery 的金色特效
- 收藏墙最后一个空位被点亮，全系列收集完成

### 玩家最常做的决策

- "我要不要再多摇几次确认？"（次数有限 vs 信息更多）
- "我觉得是这个，Pick！"（直觉判断 vs 反复对比）

### 10秒 / 1分钟 / 10分钟

- **10秒**：看到盲盒在晃、听到不同声音
- **1分钟**：摇完一局，揭晓结果，感受一个完整的猜-开循环
- **10分钟**：连开几局，收藏墙多了几个角色，开始区分不同角色的音色差异

## 四、核心玩法

### 4.1 核心循环

```text
[选一个想猜的角色] → [摇盒探测（听声+感震+看晃幅）] → [Pick 猜角色] → [揭晓结果]
       ↑                                                                    ↓
       └──────────── 收集/未收集 → 再来一局 ←─────────────────────────────────┘
```

### 4.2 玩家每局在做什么

1. **玩家主要操作**：选择目标角色 → 摇盒（SHAKE）→ 选定猜测（PICK）
2. **玩家在对什么对象做操作**：3个并排盲盒，其中一个藏了目标角色
3. **玩家每局目标**：通过探测判断哪个盒子里是自己想要的角色
4. **一局什么时候结束**：玩家点击 PICK 开盒，揭晓结果

### 4.3 基础规则

| 必须定义 | 规则 |
|---------|------|
| **操作方式** | 点击选择 + 点击摇 + 点击开盒，全程单手可操作 |
| **胜利条件** | Pick 开盒后，盒内角色与目标一致 → 命中（收集成功） |
| **失败条件** | Pick 开盒后，盒内角色与目标不一致 → 未命中（获得的角色也入收藏） |
| **资源约束** | 每日 10 次进入游戏机会；每局 10 次摇盒机会；摇完必须 Pick |
| **反馈结果** | 命中：金色特效 + 角色卡展示 + 收集点亮；未命中：普通揭示 + 角色卡 + 收集仍记录 |

### 4.4 边界规则

- **摇次用完**：SHAKE 按钮禁用，玩家只能 PICK 开盒
- **每日机会用完**：弹窗提示看广告恢复 +1 次，或等待每小时自动恢复
- **同一角色已收集**：再次命中不重复，但收藏墙仍标记已拥有
- **隐藏款出现**：3 个盒子中随机 1 个可能替换为隐藏款（Mystery），玩家无法从首页选择隐藏款为目标
- **隐藏款保底**：连续未出现隐藏款 20 局后，下次必定出现
- **已全收集**：仍可进入游戏（重复开盒），收藏墙显示 100%

## 五、玩法机制

### 机制 1：物理基因系统

| 项目 | 说明 |
|------|------|
| **机制名称** | 物理基因（Physics Gene） |
| **机制作用** | 让每个角色在盒子里创造出不同的物理状态，产生可区分的音色差异——核心推理依据 |
| **触发条件** | 角色确定时自动绑定，4 维物理参数决定 |
| **玩家应对方式** | 凭听觉/触觉/视觉差异区分不同角色 |
| **结果反馈** | 不同角色摇出不同音色、晃幅、震动模式 |
| **失败/滥用风险** | 物理参数过近的角色可能难以区分——需 playtest 确认最小可辨间距 |

**4 维物理基因**：

| 基因 | 范围 | 物理含义 | 听感影响 |
|------|------|---------|---------|
| **fill** | 0~1 | 盒内填充度（0=空 1=塞满） | 音量↓ 晃幅↓ 沉闷感↑ |
| **rattle** | 0~1 | 内容松散度（0=固定 1=散乱） | 高频脆响↑ 撞击次数↑ |
| **damping** | 0~1 | 内衬阻尼（0=裸放 1=厚包装） | 高频↓ 回响↓ 整体闷 |
| **size** | 0~1 | 角色体型（0=迷你 1=大型） | 低频↑ 音调↓ 速度↓ |

**COZY FRIENDS 系列角色物理基因**：

| 角色 | fill | rattle | damping | size | 盒内状态 | 听感描述 |
|------|------|--------|---------|------|---------|---------|
| Pip | 0.10 | 0.70 | 0.10 | 0.15 | 轻小空晃 | 清脆六面晃 |
| Mochi | 0.20 | 0.40 | 0.15 | 0.25 | 软圆微空 | 温和晃荡 |
| Biscuit | 0.40 | 0.85 | 0.15 | 0.50 | 配件咔哒 | 清脆高频咔哒 |
| Lumi | 0.35 | 0.65 | 0.10 | 0.30 | 翅膀扑腾 | 偏重+偶尔脆 |
| Crème Puff | 0.30 | 0.30 | 0.20 | 0.35 | 蓬松微脆 | 软闷微弹 |
| Daisy | 0.35 | 0.50 | 0.15 | 0.40 | 花饰轻碰 | 温和脆响 |
| Hugo | 0.92 | 0.05 | 0.30 | 0.95 | 极沉摇不动 | 几乎无声音 |
| Rex | 0.80 | 0.10 | 0.25 | 0.80 | 沉重闷响 | 低沉闷响 |
| Mystery | 0.50 | 0.20 | 0.40 | 0.55 | 神秘未知 | 闷+强回响 |

**物理基因 → 7 维音频参数公式**：

```
source   = rattle > 0.5 ? 2 : (fill < 0.3 ? 1 : 0)
pitch    = round((1-size)*8 + rattle*3 - fill*4 - 6)
speed    = 0.6 + (1-fill)*0.5 + rattle*0.3
eqLow    = round((1-fill)*10 + size*2 - 6)
eqHigh   = round(rattle*12 - damping*10 - 2)
reverb   = max(0, (1-fill)*(1-damping)*0.5)
volume   = max(0.1, (1-fill)*0.8 + rattle*0.2)
```

**扩展性**：新系列角色只需填 4 个物理参数，音色自动计算。同系列间有差异，跨系列同物理状态自然接近——因为都是盒子。

### 机制 2：探测反馈三件套

| 项目 | 说明 |
|------|------|
| **机制名称** | 探测反馈三件套（Sound + Vibration + Animation） |
| **机制作用** | 让玩家通过三种感官通道获取盲盒内部信息，是核心推理的输入 |
| **触发条件** | 玩家点击 SHAKE 按钮 |
| **玩家应对方式** | 综合"听到什么+感到什么+看到什么"判断盒内角色 |
| **结果反馈** | 音效（3段真实录音底+基因变换）+ 震动（pattern映射）+ 动画（CSS变量驱动晃幅） |
| **失败/滥用风险** | 三件套不一致会让玩家困惑——必须帧级同步 |

**音效链**：3 段真实盲盒录音 × 物理基因参数变换 = 每个角色独特音色

**震动映射**：从音效基因推导（音量→强度、pitch→频率、speed→节奏、reverb→尾震）

**动画幅度映射**：

```
fill   = 1 - volume*(0.5+speed*0.3)
loose  = reverb*0.7 + (eqLow/12)*0.3
space  = 1 - fill

--ampX = 2 + space*10 + loose*6  (px)
--ampY = fill<0.4 ? (1-fill)*8*0.7 : 2  (px)
--rot  = 1 + space*6 + loose*3  (deg)
速度  = 0.6 - space*0.18 - loose*0.08  (s)
```

**时序同步**：音效撞击点（hitTimes）→ 震动 buzz → 动画冲击放大，帧级对齐。

### 机制 3：甜区调控

| 项目 | 说明 |
|------|------|
| **机制名称** | 甜区调控（Spread + Clarity） |
| **机制作用** | 控制游戏难度：主线容易区分，挑战模式需要反复对比 |
| **触发条件** | 不同模式使用不同参数 |
| **玩家应对方式** | 主线凭直觉即可，挑战模式需要更多摇次+仔细对比 |
| **结果反馈** | 差异度大→一摇就分清；差异度小→反复摇也难分辨 |
| **失败/滥用风险** | spread=0 时无法区分，clarity=0 时同一盒两次摇不同——需限制合理范围 |

| 参数 | 主线 | 挑战 | 作用 |
|------|------|------|------|
| **spread**（差异度） | 0.85 | 0.35 | 控制不同角色基因差异大小 |
| **clarity**（清晰度） | 0.88 | 0.45 | 每次摇加随机扰动，0=同盒两次不同 |

### 机制 4：隐藏款

| 项目 | 说明 |
|------|------|
| **机制名称** | 隐藏款（Mystery） |
| **机制作用** | 制造惊喜峰值，驱动长线追角色 |
| **触发条件** | 5% 基础概率出现在 3 个盒子之一；保底 20 局未出后必出 |
| **玩家应对方式** | 无法从首页选为——只有开盒才知是否命中隐藏款 |
| **结果反馈** | 命中隐藏款：金色特效 + "SECRET" 标签 + 特殊音效；未命中：普通揭示 |
| **失败/滥用风险** | 概率过低挫败感强——保底机制兜底 |

---

### 机制 5：每日资格与恢复

| 项目 | 说明 |
|------|------|
| **机制名称** | 每日资格与恢复 |
| **机制作用** | 控制单日游戏量，制造稀缺感，留出广告位 |
| **触发条件** | 每次进入游戏页（P1首页START或P2收藏墙点击）扣 1 次 |
| **玩家应对方式** | 合理分配每日机会；看广告 +1 次；等自然恢复 |
| **结果反馈** | 首页 START 按钮显示剩余次数和恢复倒计时 |
| **失败/滥用风险** | 次数过少→无法体验完整循环；过多→无稀缺感 |

**规则**：

| 项目 | 数值 |
|------|------|
| 每日进入次数 | 10 次 |
| 自然恢复 | 每小时 +1 次 |
| 上限 | 10 次（不超储） |
| 看广告 | +1 次进入机会 |
| 每局摇盒次数 | 10 次（摇完必须 Pick） |
| Pick | 不限次（进入已扣过资格） |

### 机制 6：抽象概念盲盒

| 项目 | 说明 |
|------|------|
| **机制名称** | 抽象概念盲盒（Abstract Concept Blind Box） |
| **机制作用** | 线上对比线下的独特优势——实体盲盒只能装实物，线上不受此限，可引入抽象概念主题，天然落在判断乐趣甜区 |
| **触发条件** | 主题配置时决定——每套主题可以是实物（手办/酒瓶）也可以是抽象概念 |
| **玩家应对方式** | 对抽象内容的"感觉"更依赖直觉——微风和大风在盒子里怎么动？全凭想象+三件套反馈 |
| **结果反馈** | 抽象内容天然"相近但可分辨"，玩家靠细微差异做判断，体验更有趣 |
| **失败/滥用风险** | 抽象内容如果差异过小→纯瞎蒙；需和实物主题一样做甜区调控 |

**示例：风主题**

| 角色 | fill | 物理状态 | 听感 |
|------|------|---------|------|
| 微风 | 0.10 | 轻柔空晃 | 几乎无声的细微晃 |
| 大风 | 0.25 | 猛烈撞击 | 碎响+强震 |
| 山谷风 | 0.35 | 回响悠长 | 闷+回响 |
| 季风 | 0.50 | 周期性节奏 | 有节奏的闷响 |
| **太阳风** ⭐ | 0.15 | 神秘高频 | 极轻+极空+高频嗡鸣 |

**扩展性**：情绪主题（开心/忧愁/愤怒，隐藏款狂喜）、元素主题（火/水/土/风，隐藏款暗物质）、声音主题（钢琴/吉他/鼓，隐藏款交响乐）等，公式不变只需填 4 个物理参数。

## 六、页面与流程

### 流程图

```text
P1 首页 ──START──→ P3 游戏页 ──Pick──→ P4 结果页
  │                   ↑                     │
  └──Collection──→ P2 收藏墙 ──点击未收集──→┘
                       │                     │
                       ←───── 返回 ←─────────┘
                       （从哪来回哪去）
```

### P1 首页

| 项目 | 说明 |
|------|------|
| **进入条件** | 启动游戏 / 从其他页面返回首页 |
| **离开条件** | START → P3 游戏页；Collection → P2 收藏墙 |
| **页面内容** | 系列Logo + 当前角色展示（可左右滑动切换）+ 角色名 + START 按钮（内嵌剩余进入次数 `📦 8/10 plays · +1 in 23m`）+ Collection 按钮 |

### P2 收藏墙

| 项目 | 说明 |
|------|------|
| **进入条件** | 从 P1 点 Collection / 从 P3/P4 返回 |
| **离开条件** | 点击未收集角色 → P3 游戏页；点击已收集角色 → 放大预览；返回 → P1 |
| **页面内容** | 系列名 + 3×3 角色格子（已收集=角色卡+缩略图，未收集=? 占位）+ 收集进度条 + 预览按钮（全家福）+ 返回按钮 |

### P3 游戏页

| 项目 | 说明 |
|------|------|
| **进入条件** | 从 P1 START 或 P2 点击未收集角色 |
| **离开条件** | Pick → P4 结果页；返回 → 来源页（P1 或 P2） |
| **页面内容** | 目标角色名 + 3 个盲盒横排轮播（中间选中高亮）+ SHAKE 按钮（显示剩余摇次 `×N`）+ 3 个候选角色缩略图 + PICK 按钮 + 返回按钮 |

### P4 结果页

| 项目 | 说明 |
|------|------|
| **进入条件** | P3 点击 PICK 开盒 |
| **离开条件** | Continue / Back to Home / Back to Wall → 来源页；Play Again → P3 再来一局 |
| **页面内容** | 揭晓动画（盒子打开+角色弹出）+ 角色卡（命中=金色边框+NEW标记，未命中=普通，隐藏款=SECRET标签）+ Continue 按钮 + Play Again 按钮 + 返回首页按钮 |

## 七、主题及风格

| 维度 | 描述 |
|------|------|
| **题材/世界观** | COZY FRIENDS 系列——一群温暖治愈的小动物/小怪物，住在一个毛绒质感的梦幻世界。每个角色有独特性格，但世界观不强调叙事，重在"可爱+陪伴"的收集感 |
