# ShakeSecret 切图需求文档 · iPhone 17 Pro

> 版本 v1 · 2026-06-15 · 给设计的切图/出图规范，基于「iPhone17Pro 重制概念稿」4 张。
> 目标：路线 A —— 100% 还原概念稿，且保持游戏动态可交互（收集墙随收集变化、盒子洗牌、隐藏款逻辑等）。

---

## ‼️ 头号问题：当前切图是「假透明」（必须重导）

经检测，**当前 54 张元素切图全部 `hasAlpha: no`**——即**没有真透明通道**，AI 把透明背景画成了**真实的棋盘格像素**贴上去了。所以游戏里显示成花花的棋盘格白边。

**唯一正解：用真 alpha 重新导出 PNG-32。**

- ❌ 不要让 AI"画一个透明感的棋盘格背景"
- ✅ 必须是**真正的 alpha 通道镂空**（背景 0 不透明度）
- **自检命令**（设计/出图后逐张验证，必须回 `yes`）：
  ```bash
  sips -g hasAlpha 文件名.png      # 必须显示 hasAlpha: yes
  ```
- 背景图 `bg_*.png` 例外：它们本就是不透明整图，无需 alpha。

> ⚠️ 此问题不解决，任何视觉还原都无意义。请优先重导所有**元素类**切图（角色/盒子/框/按钮/banner/tab/图标/特效）。

## 〇、全局规范（最重要，逐条照做）

| 项 | 要求 |
|---|---|
| **画布** | **402 × 874 pt（iPhone 17 Pro 逻辑分辨率）**。整页背景按此比例出图 |
| **导出倍率** | **@3x**，即整页背景 **1206 × 2622 px**；元素尺寸下表均给 @3x 像素 |
| **格式** | **PNG-24，透明底（带 alpha）**。❌ 禁止白底/米底/深色底/场景底 |
| **一物一图** | 每个可交互元素**单独**一张，**不要**把背景、多个控件、文字画在同一张里 |
| **不烤文字** | 按钮 / banner / tab / 标签一律出**空底无文字**，文字由程序叠（次数、文案会动态变） |
| **主体居中留边** | 主体居中，四周留透明 padding；不规则物（角色/盒子）可在方形画布内居中 |
| **跨页一致** | 同一角色在「首页 / 收集墙 / 候选行 / 结果页」必须长得**一模一样**（同色同造型） |
| **命名** | 用下表英文文件名，全小写，下划线分隔；不要中文前缀 |
| **安全区** | 顶部 59pt 灵动岛、底部 34pt home 条范围内**不放关键信息**（背景可铺满） |

## 八、概念稿里发现的不一致（请设计一并修正）

1. **角色跨页不一致**：游戏页 P3 候选行里 "Lumi" 画成了**企鹅**（应为仙女）、其余角色造型也与收集墙不同。→ 所有角色四页统一为**同一形象**。
2. **盒子文案写死**："SHAKE x7"、盒上文案等是图里烤死的；程序里探测次数现为 **×10** 且会变，按钮/数字必须空底由程序叠。
3. **隐藏款**：墙上用金卡 `slot_hidden` 锁住，不能露真身；真身仅 `char_mystery_full` 在揭晓时出现。
4. **未收集**：概念稿 Rex 是灰剪影，按第二节方案处理（程序置灰，或可选出 `_silhouette`）。
