## 〇、初心与核心洞察（一切设计的源头）

**用户原话提炼：**
> "人看到一个盒子，就会猜里面是什么——好奇是人的本性。做产品就要满足人性。看到一组好看的东西，想要；获得的方式是拆盲盒；中间给点判断依据；猜成功获得，猜失败重来。"

**关键升维洞察（用户提出）：**
> "**一组盲盒装什么**是一个亮点，**用什么方法猜这组盲盒**又是一个亮点，两者主题化绑定。"
> - 风雨雷电 → 听声音匹配
> - 手办 → 看重量匹配
> - 香水 → 看色彩/轮廓匹配

**这解决了前三轮的死结**：我们曾试图用「单一猜法（听声）猜所有东西」——既不准、又单调、又被市场验证会死。现在「猜法本身成为可设计、可随主题更新的内容」，是机制的升维，不是妥协。

---

## 一、设计支柱（重构版）

1. **好奇是引擎** — 满足"看到盒子就想知道里面是什么"的最原始人性。每一组盲盒登场，第一秒就要勾起"这里面是啥"。
2. **想要才会拆** — 收集品必须"好看到让人想要"。颜值负责拉新、情感、晒图。
3. **猜法随主题轮换** — 每个主题配专属猜测方式。这是新鲜感来源，也是抄不走的护城河。
4. **失败近乎无痛** — 猜错只是"这关重来"，保底次数耗尽才触发 IAA。挫败感极低，符合休闲调性。

## 二、核心引擎：主题 × 猜法（本作的灵魂）

### 2.1 双亮点结构

| 亮点 | 内容 | 职责 | 更新频率 |
|------|------|------|----------|
| **亮点一：装什么** | 一组好看的主题收集品 | 拉新 · 情感 · 晒图（让你"想要"） | 每期换一组主题 |
| **亮点二：怎么猜** | 该主题专属的猜测方式 | 留存 · 新鲜感 · 上瘾（让你觉得"有趣"） | 随主题绑定轮换 |

> **悖论的解答**（用户问：是因为好看玩，还是因为玩法有趣玩？）：
> **两者组合，分工明确——好看负责"让你来"，玩法负责"让你留"。主题是同时点亮两者的开关。**

### 2.2 机制规格：主题猜法引擎

- **目的**：让"猜测"成为可持续产出的内容，而非一次性消耗的固定机制
- **玩家体验目标**：每开一个新主题，既有"哇这组好看"的视觉惊喜，又有"这次居然这么猜"的玩法新鲜
- **输入**：玩家进入一个主题局
- **输出**：系统加载该主题的【收集品池】+【绑定猜法】+【线索数据】
- **成功条件**：玩家能在不看教程的情况下，凭主题猜法的直觉线索做出判断
- **失败状态**：猜法太抽象/线索不可读 → 玩家变成纯蒙 → 乐趣崩塌
- **调优杠杆**：线索强度（给多少提示）、候选数量、猜法的认知负荷
- **依赖**：收集系统、图鉴系统、线索生成

### 2.3 主题 × 猜法配方库（运营内容池，可持续扩充）

| 主题 | 收集品 | 绑定猜法 | 线索载体 | 技术可行性 |
|------|--------|---------|---------|-----------|
| 风雨雷电 | 天气精灵 | 听声音 | 雷声/雨声/风声音效 | 高（合成即可） |
| 潮玩手办 | 角色公仔 | 看重量 | 数值化重量条 / 晃动幅度 | 高（纯视觉） |
| 香水 | 香水瓶 | 看色彩 + 轮廓 | 瓶身剪影 + 液体颜色 | 高 |
| 宝石矿物 | 原石/切割宝石 | 看光泽折射 | 光斑动画 / 透明度 | 中 |
| 乐器 | 迷你乐器 | 听音色 | 单音采样 | 中 |
| 动物 | 萌宠 | 听叫声 / 看影子剪影 | 叫声 / 轮廓黑影 | 高 |
| 食玩 | 微缩美食 | 看蒸汽/颜色/形状 | 视觉线索 | 高 |

> **设计原则**：优先「纯视觉线索」的猜法（重量条、轮廓、颜色、光泽）——它们不依赖耳机、不受环境干扰、答错可回看，彻底绕开"听声辨物"被验证的三个硬伤。听觉类猜法只用在"声音是该主题天然属性"的场景（雷电、乐器、动物），且作为氛围加成而非唯一判定。

## 二·五、乐趣来源与核心机制（v2.1 新增，本作真正的设计难点）

### 2.5.1 乐趣来源的精确定位（用户三条补充提炼）

| 维度 | 定位 | 含义 |
|------|------|------|
| **信息前提** | 已知品类，未知分配 | 玩家**进局前就知道**这组是哪几样东西（如"风雨雷电"4 种），**不知道的是哪个盒装哪个**。这是「N 选 N 匹配/排列」问题，不是「这是啥」的识别问题 —— 难度天花板高得多 |
| **乐趣** | 在判断的**过程** | 组内反复对比 → 缩小范围 → 权衡 → 信息不全时下注。**这是要保护的核心体验** |
| **成就** | 在猜对的**结果** | 揭晓瞬间"我赌对了" + 获得收集 + 赌徒爽感 |
| **人性钩子** | 好奇 + **赌** | 用户最初要"听声辨物"的真正原因：一组内线索相近 → 必须反复对比 → 最终下注 → 看结果。**"赌"是核心情绪之一** |

### 2.5.2 命门：早教 ↔ 劝退 的甜区（必须时刻警惕）

> 用户原话："如果设计得太简单，用户会以为是过家家，比如风雨雷电做新手引导还行，作为正式关卡就变成婴幼儿早教了。"

**判断乐趣甜区模型：**

| 区分度 | 体验 | 结果 |
|--------|------|------|
| **太大（太易）** | 一眼/一耳就分清，无需对比、无需赌 | 早教感，"过家家"，秒退 |
| **甜区（相近但可分辨）** | 要对比 2–3 次才敢押；有把握但不确定；押对 = 真本事 + 一点运气 | **有嚼劲，上瘾** |
| **太小（太难）** | 根本分不出区别，沦为纯瞎蒙 | 挫败，无成就感，劝退 |

**关键认知**：要保护的不是"声音"或"重量"这个**载体**，而是"**相近但可分辨**"这个**张力**。任何猜法，只要能制造这个张力，就成立；制造不了，再花哨也是早教。

### 2.5.3 核心机制规格：线索探测 + 排除推理 + 限次下注

> 这是不依赖具体感官载体的**通用判断骨架**，可套用到任何主题。声音、重量、轮廓只是它的"皮肤"。

- **目的**：把"对比 → 下注 → 赌 → 揭晓"做成可复用、可调难度的核心玩法
- **玩家幻想**：我是个有眼力/耳力的鉴定高手，能在信息不全时赌赢
- **输入**：一组 N 个盒子，装同品类下 N 个**相近**物品（已知有哪些，未知谁在哪）
- **输出**：每个盒子被押注一个答案，揭晓判定，押对入图鉴
- **核心动作**：
  1. **探测**：对盒子做"探测动作"（摇/翻/照/掂），获得**模糊线索**（不直接报答案）
  2. **对比**：在组内不同盒子间反复探测对比，缩小范围（单个看不准，对比才有据）
  3. **排除**：已确定的盒子帮助排除候选 → 越往后候选越少、推理越有据、张力越高
  4. **下注**：在信息不全时押注；可在揭晓前自由修改
  5. **揭晓**：整组或逐个揭晓，赌对的爽感 + 收集
- **成功条件**：玩家觉得"我是靠对比和推理赢的，不是纯运气"
- **失败状态（设计层面的"broken"）**：
  - 不用对比就能猜对 → 早教（区分度太大）
  - 对比了也分不出 → 瞎蒙（区分度太小 / 线索不可读）
  - 不需要下注就能确定 → 赌的情绪消失（信息成本太低）

### 2.5.4 维持甜区的四个设计杠杆（"巧思"所在）

| 杠杆 | 做法 | 作用 |
|------|------|------|
| **① 组内相似度** | 同组放"相近的几个"（4 种**风**的变体，而非风 vs 雷），强制对比 | 制造"相近"的张力 |
| **② 多维线索** | 单一线索不够分 → 叠加第二维度（声音 + 晃动 / 重量 + 轮廓），交叉验证 | 提升深度，避免单线索一眼穿 |
| **③ 信息成本** | 探测要花"行动"（摇/翻/照），每盒**限次** | 逼玩家权衡"再探一次还是直接押" → 这就是"赌"的来源 |
| **④ 排除法张力** | 已确定的盒帮你排除候选 → 越往后越紧张，最后一个最刺激 | 制造递增的情绪曲线 |

> **难度可动态调节**：区分度、线索维度数、探测次数随主题/关卡难度调整。新手关放宽（接近"太易"边界，保证首胜），正式关收窄到甜区中心。**这样既能用简单主题做新手引导，又不会让正式关变早教。**

### 2.5.5 落地示例：主题「四种风声」（演示"巧思"长什么样）

- **内容**：微风 / 大风 / 龙卷风 / 山谷回风 —— 都是风，但有差别（组内相近 = 杠杆①）
- **探测**：摇盒出声，限 3 次/盒。单听不准，**必须 A、B 来回对比**才听得出"这个尾音更长"（信息成本 = 杠杆③）
- **多维**：光听不够 → 叠加"盒子晃动幅度"视觉（龙卷风晃得最猛）→ 声音 + 晃动交叉验证（杠杆②）
- **排除**：确定龙卷风那盒 → 候选剩 3 → 最后一个不探测也能推出，但你敢直接押吗？（杠杆④）

> **乐趣不在"听出这是风"，而在"四种风里，靠对比和排除，赌出哪个是哪个"。** 这就从早教升级成了有脑力、有赌性的判断游戏。
>
> ⚠️ 注意：「重量/声音」都只是举例。任何主题的猜法都要经过"组内相近 + 线索模糊 + 探测限次"这道工序，不能直接报答案。手办未必靠重量——也可能靠"轮廓剪影 + 配件叮当声"等更贴合的组合，需逐主题专门设计。

## 三、基础循环

```
[看到一组好看的内容] → 「我想要」
        ↓
[系统给出该主题的判断线索]（看/听，随主题而定）
        ↓
[玩家逐个/整组做出猜测]
        ↓
   ┌──── 揭晓 ────┐
[猜中 → 获得收集]   [猜错 → 这关重来]
        ↓                ↓
[入图鉴 / 隐藏款]   [消耗 1 次保底机会]
        ↓                ↓
[开下一组]        [保底耗尽 → IAA 续命]
```

| 时间尺度 | 内容 |
|---------|------|
| **瞬时（10–30秒）** | 看到内容→读线索→猜→揭晓。反馈即时，拆盒有爽感 |
| **单局（3–5分钟）** | 完成一组主题盲盒的收集 |
| **长期（数天–数周）** | 图鉴补全 + 新主题上新 + 隐藏款追逐 + 晒图社交 |

## 四、失败机制（用户拍板）

| 必须定义 | 规则 |
|---------|------|
| **失败惩罚强度** | **近乎无痛**——猜错仅"这关重来"，不清空已收集 |
| **保底机制** | 每局/每日有 `[待测: 5 次]` 免费重试机会 |
| **保底耗尽** | 触发 IAA：看激励广告 `[待测]` 恢复重试次数 / 续命 |
| **设计理由** | 休闲调性要求低挫败；但"错多了才出广告"保证 IAA 展示来自真实卡点，自然不打扰 |

> 边界：保底次数恢复方式、是否有每日上限、广告频控 → 留待数值细化。

## 六、下一步建议

1. **选 1–2 个主题做新 demo**：建议「潮玩手办（看重量）」+「风雨雷电（听声）」各一，对比验证"视觉猜法 vs 听觉猜法"哪个更好玩——这正好用一个 demo 回答我们争论几轮的核心问题。
2. **验证目标**：玩家是否觉得"看到→想要→猜→获得"这个循环有趣且想再来；不同主题的猜法是否带来新鲜感。
3. demo 出来后再回头细化数值与商业化。
