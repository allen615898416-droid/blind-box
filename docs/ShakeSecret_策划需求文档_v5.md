# 《ShakeSecret（暂名：猜盲盒）》— 休闲游戏策划需求文档

> 版本 v5.0 · 2026-06-15 · 策划：GameDesigner
> 本文档严格遵循《休闲游戏开发 — 策划必给信息清单》十节结构编写，可直接递交开发。
> 所有数值为设计假设，标 `[待测]` 的需在原型 playtest 后定稿。
> 本阶段（立项初期）商业化从简（基础 IAA）；IP / 美术等资源依赖立项后配置，本文不展开。

---

## 设计支柱（一切决策的标尺）

1. **好奇心驱动收集 + 真实开盲盒体验** — 满足"看到盒子就想知道里面是什么"的最原始人性。每组盲盒登场，第一秒就要勾起"这里面是哪个"。选一个盒子、打开它——跟买真盲盒一模一样，是啥是啥。
2. **想要才会拆** — 收集品要好看到让人想要；收集呈现为"会长出来的格子墙"，缺口可见，集齐完整。虚拟收集凭"图鉴缺口焦虑+稀有度可见+零成本想要一切"驱动力，不靠美术堆到泡泡玛特级。
3. **猜法随主题轮换** — 每个主题配专属猜测方式，是新鲜感来源，也是抄不走的护城河。
4. **辨别是调味，不是主菜** — 主线轻松愉快，差别做明显，不挫败；核心爽感压在开盒仪式+收集满足+拆出惊喜。挑战关才给想烧脑的人上难度。
5. **小赌怡情** — 信息不全时下注、选一个盒子、揭晓。"赌"是核心情绪，但失败近乎无痛——开出来不是目标也获得角色。
6. **主观感知，拒绝量化** — 探测反馈必须是"需玩家主观感受"的现象（声音/震动/动画幅度），绝不替玩家算好读数（关键词/波形/数值条=早教=杀死探索）。

## 一、游戏是什么

| 信息 | 内容 |
|------|------|
| **品类** | 收集 + 解谜（全年龄向，非低龄）。子类型：盲盒收集 / 轻辨别 |
| **一句话** | 看到一组好看的东西 → 摇一摇凭直觉感受 → 选一个盒子打开 → 是啥是啥，开出目标就点亮图鉴 |
| **竞品参考** | ① Pokémon TCG Pocket（收集+图鉴+晒卡，年收 $6.85 亿，证明收集驱动可上十亿）— 学其收集内核<br>② 泡泡玛特线下/Pop Now（摇盒开箱仪式感、隐藏款）— 学其情感外壳与真实开盲盒体验<br>⚠️ ASMR Mystery Blind Bag 等"摇盒听声"类经核查月收入<$100、几乎不赚钱，作为**反面教材**：摇盒/听声只能当外壳，不能当判定核心 |
| **目标平台** | 阶段一：Web 原型（验证核心循环是否好玩）；阶段二：独立手游 App（iOS + Android） |
| **屏幕方向** | 竖屏（单手、碎片化） |
| **目标市场** | 首发泰国 + 东南亚，第二批欧美。语言：英文 + 泰文起步 |
| **变现模式** | 基础 IAA（广告变现），无付费墙、无内购货币（本阶段） |

### 核心创新点（与所有竞品的根本差异）

- **主题 × 猜法双亮点引擎**：一组盲盒"装什么"是一个亮点（收集内容），"用什么方法猜"是另一个亮点（玩法），二者主题化绑定、随主题轮换。
- **真实开盲盒体验**：选一个盒子→点"Pick it"→打开→直接看到内容。跟买真盲盒一模一样——买了就买了，是啥是啥。不再是"整组开盒"或"押中几个"。
- **线上内容可抽象**：不受实体限制，可收集"四种风/五种情绪/七种和弦"等抽象内容；抽象内容天然"相近但可分辨"，正好落在判断乐趣甜区。线上收集还能"拼成整体"——实体给不了这种完整感。
- **主观感知驱动**：探测反馈不是"读题"而是"凭感觉"——声音/震动/动画幅度，玩家要自己去感受、去对比，不是替玩家算好读数。

### 实测验证数据（已进入设计）

> 用户实玩 demo 反馈："摇了几次都没找对，有乐趣但有点难，三四次过不去就想弃游。"
> → 验证：核心机制方向成立（有乐趣），但差别做太小会掉进"太难=瞎蒙"区导致弃游。
> → 结论：**主线必须保证轻松首胜，差别做大**。这是数值问题，不是机制问题。

## 二、核心循环

```
[想补全收集墙 / 好奇这组是哪个]
        ↓
[在收集墙点击一个想要的藏品（如"龙卷风"）进入对应关卡]
        ↓
[关卡页：看到一组相近的盒子（已知品类、未知分配）]
        ↓
[探测：摇盒子 → 感受声音+震动+动画幅度 → 组内反复对比]（总探测限次 ×7）
        ↓
[选一个盒子 → 点"Pick it" → 打开盒子 → 直接看到内容]
        ↓
   ┌──── 结算 ────┐
[开出来是目标角色]      [开出来不是目标角色]
        ↓                  ↓
[完成目标，点亮目标格]   [获得该角色，点亮对应图鉴格（如未收集）；未完成本关目标]
        ↓                  ↓
[回收集墙 / 进下一关]   [本关重开（无保底、无惩罚、不清空已有）]
```

**策划必须说清的 4 件事：**

1. **玩家每局在做什么** — 摇盒子凭直觉感受、组内对比，选一个盒子打开
2. **什么时候结束** — 玩家点"Pick it"，打开选中的盒子，直接看到内容
3. **赢了给什么** — 开出目标角色 → 点亮收集墙目标格；集齐整墙 = 通关该主题
4. **输了怎么办** — 开出非目标角色 → 获得该角色（点亮对应图鉴格如未收集），本关目标未完成可重开。**无保底、无体力扣除、不清空已收集**。重开时探测次数回满、视为本关重新开始

| 时间尺度 | 内容 |
|---------|------|
| **瞬时（10–30秒）** | 摇→感受→对比→选盒→Pick it→揭晓。拆盒有爽感，开出想要的角色有多巴胺 |
| **单局（1–2分钟）** | 完成一次开盒，拿到一件角色 |
| **长期（数天–数周）** | 收集墙逐格点亮、集齐主题、追隐藏款金格、新主题上新、晒墙社交 |

## 三、规则（核心机制规格）

### 机制 1：探测（局内信息获取——凭主观感知）

| 必须定义 | 规则 |
|---------|------|
| **触发（手机）** | 摇晃（`DeviceMotionEvent`）/ 点击盒子触发探测 |
| **触发（Web）** | 点击盒子 / "摇一摇"按钮模拟 |
| **探测反馈（三件套，已拍板）** | **声音 + 震动 + 动画幅度**，三者同时给出，凭直觉感受。具体规格见《摇盒探测反馈三件套_设计规格_v1.md》 |
| **线索性质** | 探测给**现象级反馈**，需玩家主观感知，绝不直接报答案（如"这盒晃得猛"，而非"重量 ●●●○○"） |
| **探测限次** | **总探测次数** `[待测: 主线7次, 挑战5次]`，跨所有盒子共享。**限次是无保底体系下"赌性"的唯一来源** —— 逼玩家权衡"再探一次还是直接选" |
| **对比机制** | 可在选盒前自由切换探测任意盒子、反复对比，总探测次数受限 |
| **额外探测** | 次数用完可看激励广告 +探测 `[待测: 2 次]`（见七） |

#### 机制 1.1：探测反馈三件套（核心呈现，开发须知）

> 探测一个盒子时，**同时**给三种感官反馈，让玩家"凭感觉"判断，而不是读数。
> **完整技术规格见《摇盒探测反馈三件套_设计规格_v1.md》**，下表为摘要：

| 反馈通道 | 怎么表现 | 作用 |
|---------|---------|------|
| **声音** | 摇晃时播放该物品的音效（基于真实录音+7维基因参数实时变换） | 戴耳机的主通道 |
| **震动** | 手机震动马达配合摇晃节奏（强/弱/连续/顿挫），从音效基因推导 | 触觉补充，增代入 |
| **动画幅度** | **盒子本身**摇晃/颤动的幅度与节奏（重物晃得沉/大、碎物抖得碎/快），从音效基因推导 | **不戴耳机时的主通道** |

> ⚠️ 开发须知：动画幅度不是装饰，是核心线索通道之一，必须做出可分辨的差异。主线差别做大（一摇就能感觉出不同），挑战关差别做小（要反复对比品）。

#### 机制 1.2：设计红线（已验证的"死路"）

> 以下方案经实测/推演后被否决，不得使用：

| ❌ 被否方案 | 否决原因 |
|-----------|---------|
| **关键词/标签**（如"呼啸 ●●●○○"） | 直接给出量化读数=替玩家判断完=做题=早教=杀死探索乐趣 |
| **波形/频谱图** | 波形也是量化（越剧烈风越大谁都懂），同样杀死探索变读数 |
| **数值条**（重量条/强度条） | 同上，替玩家算好的东西都杀探索 |
| **任何"替玩家感知"的抽象表示** | 要保护的是"需玩家主观感知"的探索过程，任何替玩家算好的读数都会让辨别变早教 |

### 机制 2：选盒与押注（局内决策，与探测交织）

| 必须定义 | 规则 |
|---------|------|
| **交织** | 探测与选盒在**同一个核心游戏页**完成，可边探边选，无页面跳转 |
| **选盒** | 从候选盒子中选择一个盒子（3 盒 carousel，中间大两侧小+半透明） |
| **押注** | 从候选角色中选择一个角色，押给选中的盒子；提交前可任意修改 |
| **候选项** | 该组的已知品类列表（如风组的 4 种风） |

### 机制 3：揭晓与判定

| 必须定义 | 规则 |
|---------|------|
| **揭晓时机** | 玩家点"Pick it" |
| **揭晓方式** | **打开选中的盒子，直接展示内容（真实购买体验）** |
| **反馈信息** | **直接展示盒子内容，是啥是啥**；主线直接展示角色；挑战关可选"先给轮廓再揭晓"等变体 |
| **成功判定** | 开出来是目标角色 = 完成目标（点亮目标格）；非目标 = 获得该角色并点亮对应图鉴格（如未收集），但未完成本关目标 |

### 机制 4：单局玩法类型（按主题区分 + 难度递进 + C 路线）

> **C 路线（已拍板）**：主线轻松愉快（辨别只过过脑、差别做明显、不挫败），核心爽感压在开盒仪式+收集满足+拆出惊喜；另开挑战关给想烧脑的人（差别细微）。

| 玩法 | 玩家要做什么 | 成功条件 | 适用阶段 | 差异明显度 |
|------|-------------|---------|---------|-----------|
| **找出指定** | 在一组相近物里定位 1 个指定目标 | 开出目标角色 | **前期 / 主线**（门槛低、节奏快） | **大**（一摇就感觉到） |
| **多选一** | 从更多盒子中找出目标（5 盒+） | 开出目标角色 | **中期 / 挑战**（选择更多、探测更紧） | **中~小**（需反复对比） |
| **限定主题** | 特定主题活动限定角色，限时可遇 | 开出目标角色 | **活动**（限时稀有、FOMO 驱动） | **按活动配置** |

> **同一主题内玩法统一**，玩家进主题前就知道这次怎么玩，避免认知混乱。难度沿"找出指定→多选一→限定主题"递进。

### 机制 5：判断乐趣甜区（6 个调节旋钮 —— 设计的核心命门）

> 目标：主线维持在"轻松可辨"，挑战关维持在"相近但可分辨"。太易=早教秒退，太难=瞎蒙劝退。

| 旋钮 | 做法 | 作用 |
|------|------|------|
| ① 差异明显度 | 主线差别大（一摇感觉出），挑战差别小（要反复对比） | **C 路线核心旋钮** |
| ② 组内相似度 | 同组放相近的几个（4 种风的变体，而非风 vs 雷） | 强制对比 |
| ③ 多维线索 | 单线索不够分→叠加第二维度（声音+晃动幅度 / 撞击声+震动节奏） | 防一眼穿、加深度 |
| ④ 探测次数 | 总探测限次（跨所有盒子共享） | 制造信息成本→"赌"的来源 |
| ⑤ 排除法张力 | 已排除的盒帮助缩小范围 | 越往后越紧张，最后一次探测最刺激 |
| ⑥ 揭晓方式 | 主线直接展示角色；挑战关可选"先给轮廓再揭晓"等变体 | 控制揭晓节奏与惊喜感 |

### 机制 6：隐藏款（随机触发，进场当干扰项）

| 必须定义 | 规则 |
|---------|------|
| **触发** | 进入**任意关卡**时，按概率判定本关是否"混入隐藏款" `[待测: 5%/关]` |
| **明牌提示** | 一旦混入，进场即明确提示"⚡ 这关藏了隐藏款！"（点燃贪心与期待） |
| **形态** | 隐藏款作为一个**额外的未知干扰项**进场，本关多一个盒、候选多一项，**难度提升** |
| **获得方式** | 开出来是隐藏款 = 直接获得（不二次 roll）。运气好撞上隐藏款，开出来就是你的 |
| **非目标开出** | 开出来不是目标角色 = 没完成本关目标，但获得该角色并点亮对应图鉴格（如未收集） |
| **防干涸保底（已拍板）** | 玩家连续多关未遇到隐藏款（`[待测: 20 关]`）→ 概率递增至必出，避免非酋长期遇不到而流失 |
| **图鉴呈现** | 收集墙中是一个**金色格子**：进主题即可见（标★但锁定），普通格集齐 ≠ 完美，金格一直勾着玩家 |

### 机制 7：实物主题也能做（已平反）

> 用户实测反驳了"手办/酒只能靠看脸、做不了"的武断结论：

| 实物类型 | 怎么做辨别 | 原理 |
|---------|-----------|------|
| **手办/公仔** | 形状不同→撞击盒壁声不同 / 重量不同→晃动幅度不同 / 接触面不同→震动节奏不同 | 三件套天然适用 |
| **酒瓶/容器** | 液体晃荡声不同 / 重量不同→晃动幅度 / 碰撞音不同 | 同上 |

> 关键认知：**不是"做不了实物主题"，是要换思路——不靠"看长相"区分，靠"它在盒子里怎么折腾"区分。** 这跟风的逻辑是同一个：让盒里的东西通过"动静"暴露自己。

### ⚠️ 必须定义的边界情况（不定义开发一定会猜）

| 边界情况 | 裁定 |
|---------|------|
| 玩家一次都不探测直接选盒 | 允许（等于纯蒙） |
| 开出非目标角色后玩家重开 | 重开探测次数回满，视为本关重新开始；已获得的角色不清空 |
| 隐藏款混入但玩家没察觉 | 已明牌提示，不会发生；若关闭提示则按暗牌彩蛋处理（默认明牌） |
| 探测音效/动画播放中切换盒子 | 立即停止当前、播放新盒（不叠加） |
| 激励广告加载失败 | 不扣任何东西，直接给奖励或静默跳过，绝不"白等" |
| 玩家中途退出未结算 | 进度不保存，下次重开本关（休闲不做断点续玩） |
| 配额用完但想继续开盒 | 看激励广告获取额外配额，或等待回血（见第五节配额机制） |

## 四、页面与流程

### 页面流转

```
启动 → P1主页 → P2收集墙(选系列+选藏品) → P3核心游戏页(摇/选盒/Pick it) → P4结果页
                  ↑__________________________________________|
                         (回收集墙 / 继续下一关)
```

### 4 个核心页面结构定义

> 竖屏手机。美术风格：潮玩盲盒萌系 IP，往泡泡玛特审美靠（大胆学风格；不复制 Labubu 具体形象/不用其商标）；糖果奶油色系；全年龄、有质感、不幼齿。

#### 页面 1 · 主页

**作用**：进入 App 的家，传达"想玩、有品质"。

**区域结构**：
1. 顶部条：游戏 Logo + 设置入口
2. 主视觉区（中部）：当期主推主题的明星藏品大图 + **START** 主按钮
3. 底部：**Collection Guide** 入口（跳转收集墙）

| 页面 | 进入条件 | 离开条件 | 页面上有什么 |
|------|---------|---------|-------------|
| **主页** | 启动后 | 点 Collection Guide / START / 设置 | Logo、主推主题、START 按钮、Collection Guide 入口 |

#### 页面 2 · 收集墙

**作用**：核心驱动页。看见"拥有/缺什么"，选想要的进关。**缺口要勾人。**

**区域结构**：
1. ① **系列 Tab 栏**：横向可滑的主题标签/卡片
2. ② **系列名行 + 👁 预览**：当前系列名称 + 预览入口
3. ③ **3×3 网格**：当前主题的收集墙
   - 已得格：点亮、显示藏品
   - 未得格：灰暗 + 问号「?」（缺口感）
   - 隐藏款格：金色 + 星标★（锁定时也可见，勾人）
4. ④ **进度条 + SHARE**：该主题收集百分比 + 晒图/分享按钮

| 页面 | 进入条件 | 离开条件 | 页面上有什么 |
|------|---------|---------|-------------|
| **收集墙** | 主页/结算 | 点某藏品进关 / 返回 | 系列Tab栏、系列名行+👁预览、3×3网格(灰格?/彩格/金格★)、进度条、SHARE |

#### 页面 3 · 核心游戏页（最重要，80% 体验）

**作用**：摇盒、感受反馈、组内对比、选盒开盒。**所有动作一页完成，不跳页。**

**区域结构（从上到下，固定骨架）**：
1. **find 目标区**：`find: [目标角色轮廓?] (0/1)`；若隐藏款触发，此处有醒目「⚡隐藏款」提示
2. **3 盒 carousel**：3 个盒子，中间大两侧小+半透明，选中的高亮
3. **反馈区**：选中盒子摇晃时，盒子在此**颤动/摇晃**（动画幅度=线索）；配声音、震动。**不放波形/数值/关键词**
4. **操作区**：「SHAKE」按钮（显示剩余总次数 ×7）+ 候选角色行 +「Pick it」按钮

| 页面 | 进入条件 | 离开条件 | 页面上有什么 |
|------|---------|---------|-------------|
| **核心游戏页** | 收集墙点击某藏品 | Pick it / 退出 | find目标区+隐藏款提示、3盒carousel、反馈区(三件套)、SHAKE按钮×7、候选行、Pick it |

> **核心游戏页是整个游戏 80% 的体验所在**，探测与选盒同框、信息与决策不分离，是首要打磨对象。

#### 页面 4 · 结果页

**作用**：开盒的高潮 + 收获满足。情绪爆发点。

**区域结构（全屏覆盖层）**：
1. **一个盒子**：选中的盒子打开，展示角色
2. **结果状态**：`Got [角色名]!` / `NEW!`（首次获得）/ `Already owned`（已有）
3. **隐藏款解锁时**：金色特别揭晓（闪光、星星，最显眼）
4. **主按钮**：**Continue**（文案按场景变："Next Round" / "Back to Wall" / "Keep Going" 等）

| 页面 | 进入条件 | 离开条件 | 页面上有什么 |
|------|---------|---------|-------------|
| **结果页** | Pick it | Continue | 盒子打开、Got/NEW/Already owned、隐藏款金色揭晓、Continue 按钮 |

|----------|--------|--------|---------|-----------|

### 配额机制详细规格

| 必须定义 | 规则 |
|---------|------|
| **叙事包装** | "店铺每日库存"——每天到货一批盒子，卖完等补货 |
| **免费配额** | 每日 `[待测: 8 次]` 免费开盒 |
| **回血速率** | 每 `[待测: 90 分钟]` 回复 1 次配额 |
| **广告补充** | 看激励广告 +1 次配额 |
| **广告冷却** | 广告补充后 `[待测: 30 秒]` 冷却才可再看 |
| **硬上限** | **无硬上限**（可通过广告无限获取，但冷却限制刷速） |
| **配额上限** | 累计不超过 `[待测: 12 次]`（防囤积） |
| **刷新时间** | 每日 `[待测: 04:00 本地时间]` 重置免费配额至满 |

## 六、经济（IAA 模式，核心是"收集进度"而非货币）

```
玩家获得的（价值 = 收集进度，非货币）：
  - 开出目标角色 → 完成本关目标，点亮收集墙目标格
  - 开出非目标角色 → 获得该角色，点亮对应图鉴格（如未收集）
  - 集齐普通格 → 主题接近完成
  - 解锁隐藏款金格 → 主题完美通关
  - 看激励广告 → 额外探测 / 额外配额

玩家花费的（注意力，非货币）：
  - 看激励广告（自愿）
  - 看插屏广告（被动，控频）
  - 配额（每日商店库存）

控制（防破坏体验 / 防广告疲劳）：
  - 单次激励"额外探测"每关最多触发 [待测: 1 次]
  - 插屏每 [待测: 2-3 关] 一次，单关内绝不插
  - 新手前 [待测: 3 关] 不出任何广告（保护首次体验）
  - 隐藏款防干涸保底，避免收集进度卡死
  - 配额回血确保免费玩家也能持续玩，广告为加速而非必需
```

> 设计理由：本作的"经济"本质是**收集进度的供需**——角色是"源"，图鉴缺口是"汇"，缺口焦虑驱动玩家持续开盒。配额系统（"每日商店库存"）制造稀缺感与期待感，但回血机制保证免费玩家不被卡死。IAA 收入来自"额外探测/配额补充"这类**主动激励视频**（既提留存又产生广告展示）+ 控频插屏。

## 七、广告与变现（基础 IAA）

| 广告位 | 类型 | 什么时候展示 | 看广告的奖励 | 加载失败怎么办 |
|--------|------|-------------|-------------|---------------|
| **额外探测** | 激励视频 | 总探测次数用完、想再探 | +2 次探测 | 直接给 +2（不让白等） |
| **配额补充** | 激励视频 | 配额用完想继续开盒 | +1 次开盒配额 | 直接给 +1 |
| **隐藏款概率提升**（可选） | 激励视频 | 进关前 | 提升下一关隐藏款概率 | 用基础概率 |
| **插屏广告** | 插屏 | 每 2-3 关之间（绝不在关内） | 无（标准 IAA） | 静默跳过 |

**统一原则：广告加载失败时永远倾向"给好处 / 静默跳过"，绝不出现"白等"或"没看也给奖"的逻辑漏洞。**

## 八、关卡递进（主题与玩法双轴递进 + C 路线）

| 必须说清 | 内容 |
|---------|------|
| **总量 / 更新** | 首发 `[待测: 5 套主题]`，之后每周/每两周更新 1 套（运营驱动留存） |

| **每套引入什么新东西** | 第 2 套：隐藏款机制；第 3 套：多维线索；第 4 套：限定节日主题（泼水节等）；第 5 套：多选一挑战玩法（5 盒+更少探测） |
| **手配 or 算法** | **手配**。每套主题需配置：藏品池、绑定猜法、线索数据、隐藏款、网格布局 → 需要"主题配置表 + 关卡编辑器" |

## 十、范围边界

| 做 | 不做 |
|---|------|
| 主题×猜法双亮点核心玩法 | 不做实时多人对战（首版） |
| 3×3 网格收集墙 + 隐藏款金格 | 不做写实画卷为主线（成本+反噬，仅旗舰特殊用） |
| 真实开盲盒：选一个盒子→Pick it→开出啥是啥 | 不做整组开盒 / 押中几个判定（v4 旧模型，已废弃） |
| 单局玩法递进（找出指定 / 多选一 / 限定主题） | 不做同一主题内混合玩法（认知负荷，留作后期进阶） |
| 主线轻松+挑战烧脑（C 路线） | 不做全线高难辨别（实测验证会弃游） |
| 探测三件套：声音+震动+动画幅度 | **不做关键词/波形/数值条**（量化=早教=杀死探索） |
| 基础 IAA | 不做内购货币 / 付费抽卡（避赌博合规风险） |
| 原创潮玩盲盒萌系 IP（往泡泡玛特风格靠） | **绝不使用 Labubu / Pop Mart / Molly 等商标**（侵权红线） |
| 手配主题 + 周更 + 关卡编辑器 | 不做算法生成关卡（首版） |
| 失败本关重开、无保底 | 不做传统体力系统；使用"每日商店库存"回血式配额 |
| 隐藏款随机触发"遇到"、开出来即得 | **不做隐藏款二次 roll**（开出来就是你的，绝不"赢了还不给"） |
| 配额系统：回血式 + 广告补充 + 无硬上限 | 不做断点续玩 / 云存档（首版本地存储） |

## 附：一句话总结

**玩家在收集墙上看到一个想要的藏品，点进去面对几个相近的盒子，摇一摇凭直觉感受哪个是它，选一个打开——开出目标就点亮图鉴，开出别的也收入囊中；运气好还会撞上明牌的隐藏款，开出来就是你的。主线轻松愉快，差别做明显，核心爽感在开盒+收集+拆出惊喜；挑战关才上硬菜，差别细微、要反复品。每一刻在"探-比-选"，每一关在"赌自己的直觉"，长期在"把一面墙点亮"。**
