# 《Shake & Guess（盲盒摇摇乐）》— 休闲游戏策划需求文档

> 版本 v0.1 · 2026-06-13 · 作者 GameDesigner
> 本文档遵循《休闲游戏开发 — 策划必给信息清单》模版编写，目标是让开发无需猜测即可动工。
> 所有数值均为设计假设，标记 `[待测]` 的需在原型 playtest 后定稿。

---

## 设计支柱（一切决策的标尺）

1. **听得出，才赢得了** — 声音是唯一的判定依据，不是装饰。玩家的耳朵越练越准，这是技巧深度的来源。
2. **小赌怡情，输不伤心** — 失败是"再听一次的挑战"，不是清零惩罚。整组判定 + 打乱重听，张力来自"我到底听懂没有"。
3. **收集是终点，听声是路径** — 图鉴补全提供长期目标；摇盒听声提供每一刻的乐趣。
4. **零门槛上手，高手有天花板** — 新手靠"差异大"的声音轻松入门，老手能分辨同系列的细微差别。

## 一、游戏是什么

| 信息 | 内容 |
|------|------|
| **品类** | 休闲收集 + 音频解谜（听声辨物）。子类型：盲盒收集 / ASMR 解压 |
| **竞品参考** | ① ASMR Mystery Blind Bag（SkyRise Games，全球 100万+ 下载，主打 ASMR 开箱，但**结果随机、听声不参与判定**）<br>② 拉布布神秘盲盒（蹭 Labubu IP，点击撕包随机掉落）<br>③ Mystery Sound Box（真·听声音猜物品的派对游戏，验证"听声猜"可玩）<br>**差异化**：以上无一款把"听声判断→玩家决策→决定成败"做成核心，我们做这块空白 |
| **目标平台** | 阶段一：Web 原型（验证核心假设）；阶段二：独立手游 App（iOS + Android） |
| **屏幕方向** | 竖屏（单手操作，贴合摇晃手势与碎片化场景） |
| **目标市场** | 首发泰国 + 东南亚（印尼/马来/菲律宾/新加坡），第二批欧美。语言：英文 + 泰文起步 |
| **变现模式** | IAA（广告变现），无付费墙，无内购货币购买 |

## 二、核心循环

```
[想补全图鉴 / 想再赢一组] 
        ↓
[开启一套主题盲盒（6 个未知盒排成一排）]
        ↓
[逐个：摇盒 → 听声 → 可对比摇其他盒 → 再听 → 给该盒押一个答案]
        ↓
[6 个全押完 → 结算整组正确率]
        ↓
   ┌────分支────┐
[≥60% 成功]      [<60% 失败]
   ↓                ↓
[整组收入图鉴]   [盒子位置打乱，重新听一轮]
   ↓                ↓
[触发隐藏款判定]  [回到逐个押注]
   ↓
[回到首页 / 开下一套]
```

**策划必须说清的 4 件事：**

1. **玩家每局在做什么** — 对一排盲盒逐个"摇—听—对比—押注"，最后整组判定
2. **什么时候结束** — 6 个盒全部押注完毕，触发整组正确率结算
3. **赢了给什么** — 整组盲盒收入图鉴；连续成功累计"耳力经验"；可能触发隐藏款
4. **输了怎么办** — 盒子位置打乱，物品不变，重新听一轮（不清零、不扣体力）。可选：看广告"复活重押"挽救临界局

## 三、规则（核心机制规格）

### 机制 1：单盒"摇—听—押"

| 必须定义 | 规则 |
|---------|------|
| **触发摇晃（手机）** | `DeviceMotionEvent` 加速度超过阈值 `[待测: 15 m/s²]` 判定为一次有效摇晃 |
| **触发摇晃（桌面/Web）** | 长按盒子 + 鼠标快速移动 模拟摇晃；或点击"摇一摇"按钮 |
| **播放声音** | 每次摇晃触发该盒对应物品的音效（Web Audio 实时合成或预录音频） |
| **摇晃次数限制** | 每盒免费可摇 `[待测: 3 次]`；用完后可看激励广告 +摇 `[待测: 2 次]`（详见七） |
| **对比机制** | 玩家可在押注前自由切换摇任意盒子，反复对比，无次数惩罚（仅单盒摇晃次数受限） |
| **押注** | 从该套主题的候选物品列表中，给当前盒选 1 个答案；可改，直到点"确认整组" |

### 机制 2：整组正确率判定

| 必须定义 | 规则 |
|---------|------|
| **判定时机** | 6 个盒全部押注 + 玩家点"确认整组" |
| **成功线** | 正确率 ≥ 60%（6 个中对 4 个及以上）→ 整组成功 |
| **成功结果** | 整套盲盒全部收入图鉴（含押错的盒，按真实物品入库）；累计耳力经验 |
| **失败结果** | 盒子位置**随机打乱**（物品不变），玩家重新听一轮、重新押注。本套不入库直到成功 |
| **失败次数上限** | 无硬上限（休闲友好）；但每次失败后可触发"看广告复活重押"作为捷径 |

### 机制 3：隐藏款

| 必须定义 | 规则 |
|---------|------|
| **触发条件** | 整组成功后，按概率判定本套是否含隐藏款 |
| **基础概率** | `[待测: 5%]` per 成功整组 |
| **广告增益** | 看激励广告 → 本次隐藏款概率翻倍至 `[待测: 10%]`（详见七） |
| **隐藏款价值** | 图鉴单独栏位，视觉特效区分，社交炫耀点 |

### ⚠️ 必须定义的边界情况（不定义开发一定会猜）

| 边界情况 | 裁定 |
|---------|------|
| 玩家一次都不摇就直接押注 | 允许（等于纯蒙），但不给"耳力经验"加成 |
| 同一套里两个盒装了同样的物品 | 允许；候选列表里该物品可被押多次 |
| 失败打乱后，玩家发现某盒上次押对了 | 打乱后玩家不知道哪个是哪个（位置变了），需重新听。上次的押注记录清空 |
| 正确率刚好卡在临界（如对 3 个 = 50%） | 判失败（<60%）；此时优先弹"看广告复活重押 1 个"挽救 |
| 摇晃音效正在播放时玩家切换到别的盒 | 立即停止当前音效，播放新盒音效（不叠加） |
| 激励广告加载失败 | 不扣任何东西，直接给奖励或跳过（详见七），绝不"白等" |
| 玩家中途退出未结算的局 | 进度不保存，下次重新开这套（休闲游戏不做断点续玩，降低复杂度） |

## 四、页面与流程

```
启动页 → 首页 → 主题选择 → 游戏中（摇-听-押） → 整组结算
                  ↓            ↓                    ↓
                图鉴         设置/复活弹窗        隐藏款揭晓
                  ↓
                成就 / 耳力等级
```

| 页面 | 进入条件 | 离开条件 | 页面上有什么 |
|------|---------|---------|-------------|
| **启动页** | App 打开 | 加载完成自动进首页 | Logo、加载进度 |
| **首页** | 启动后 | 点主题/图鉴/设置 | 当前主推主题、耳力等级、图鉴进度条、每日任务入口 |
| **主题选择** | 首页点"开始" | 选定一套 / 返回 | 主题列表（部分需看广告解锁，详见七）、每套图鉴完成度 |
| **游戏中** | 选定主题 | 整组结算 / 退出 | 6 个盲盒、摇晃手势区、候选物品列表、剩余摇晃次数、"确认整组"按钮 |
| **整组结算** | 押注完成 | 继续/返回首页 | 正确率、每盒对错、入库动画、耳力经验、隐藏款揭晓 |
| **复活弹窗** | 失败且临界 | 看广告/放弃 | "看广告复活重押 1 个" / "重新听一轮" |
| **图鉴** | 首页/结算 | 返回 | 所有物品收集格、隐藏款专栏、收集百分比、晒图分享按钮 |
| **设置** | 首页 | 返回 | 音量、震动开关、语言、隐私政策 |

## 五、道具 / 特殊机制

| 道具 | 做什么 | 从哪来 | 消耗什么 | 用完怎么办 |
|------|--------|--------|---------|-----------|
| **额外摇晃** | 单盒多摇 2 次 | 看激励广告 | 1 次广告 = +2 摇 | 可再看广告 |
| **复活重押** | 失败时重押 1 个盒挽救整组 | 看激励广告 | 1 次广告 = 1 次重押 | 只能"重新听一轮" |
| **隐藏款增益** | 本组隐藏款概率翻倍 | 看激励广告 | 结算前看 1 次广告 | 用基础概率 |
| **耳力等级（非道具，成长系统）** | 解锁辅助：如"声纹可视化提示""候选范围缩小" | 累计听声经验自动升级 | 不消耗 | 持续成长 |

> 注意：本游戏 **无内购货币**。所有"道具"均通过激励广告获得，契合纯 IAA 模式。

## 六、经济（IAA 模式，无货币系统）

```
玩家获得的（价值 = 进度/便利，非货币）：
  - 整组成功 → 整套物品入库
  - 看激励广告 → 额外摇晃 / 复活重押 / 隐藏款翻倍
  - 每日登录 → 解锁今日限定主题试玩
  - 完成每日任务（如"成功收集 3 套"）→ 解锁新主题

玩家花费的（注意力，非货币）：
  - 看激励广告（自愿）
  - 看插屏广告（被动，控频）

控制（防止破坏体验 / 防止广告疲劳）：
  - 单盒激励广告"额外摇晃"每盒最多触发 [待测: 1 次]
  - 插屏广告每 [待测: 2-3 局] 一次，单局内绝不插
  - 隐藏款翻倍每整组只能看 1 次广告
  - 新手前 [待测: 3 局] 不出任何广告（保护首次体验）
```

> 设计理由：IAA 命脉 = eCPM × 人均广告展示次数 × 留存。"额外摇晃""复活重押"是**主动激励视频**，既提升留存又自然产生广告展示，是核心收入来源；插屏是补充，必须控频以保留存。

## 七、广告与变现

| 广告位 | 类型 | 什么时候展示 | 看广告的奖励 | 加载失败怎么办 |
|--------|------|-------------|-------------|---------------|
| **额外摇晃** | 激励视频 | 单盒摇晃次数用完、玩家想再听 | +2 次该盒摇晃 | 直接给 +2 摇（不让白等） |
| **复活重押** | 激励视频 | 整组失败且为临界局（如 50%） | 重押 1 个盒，重新判定 | 跳过，进入"重新听一轮" |
| **隐藏款翻倍** | 激励视频 | 整组成功、隐藏款判定前 | 隐藏款概率翻倍 | 用基础概率判定，照常揭晓 |
| **主题解锁** | 激励视频 | 主题选择页，锁定主题 | 解锁该主题 1 次游玩 | 提示稍后重试，不解锁 |
| **插屏广告** | 插屏 | 每 2-3 局之间（绝不在单局内） | 无（标准 IAA） | 静默跳过，直接进下一界面 |

**统一原则：广告加载失败时，永远倾向于"给玩家好处 / 静默跳过"，绝不出现"白等 30 秒""没看也给奖励的逻辑漏洞"。**

## 八、关卡递进（主题与内容更新）

| 必须说清 | 内容 |
|---------|------|
| **总量 / 更新** | 首发 [待测: 5 套主题]，之后每周/每两周更新 1 套（运营驱动留存） |
| **难度递增** | ① 物品音效差异从"大"到"小"（新手套：金属 vs 毛绒；高手套：3 种相似金属）<br>② 每套候选物品种类增多（首套 4 种 → 后期 8 种）<br>③ 引入"干扰盒"（混入背景噪音的盒） |
| **每套引入什么新东西** | 第 2 套：隐藏款机制；第 3 套：干扰音；第 4 套：限定节日主题（泼水节/水灯节）；第 5 套：相似音辨识挑战 |
| **手配 or 算法** | **手配**。每套主题的物品池、音效、隐藏款由策划手工配置 → 需要一个简单的"主题配置表"（物品名/音效ID/稀有度/出现权重） |

## 九、视觉方向

| 必须说清 | 内容 |
|---------|------|
| **风格参考** | "丑萌"潮玩风（贴合 Labubu 审美热度），扁平 + 圆润，明亮治愈色 |
| **主色调** | 马卡龙糖果色系（粉/薄荷绿/奶黄），单局背景纯净不抢声音注意力 |
| **氛围** | ASMR 解压感：盒子有微弹动画、摇晃有粒子、入库有满足感反馈 |
| **资源规格上限** | 单套主题美术 < [待测: 20MB]；音效优先 Web Audio 合成或小体积 ogg，单音效 < [待测: 100KB] |
| **音效设计（命根子）** | 见下表，新手套差异大、高手套差异小 |

### 音效设计表（原型成败的命根子）

| 物品类型 | 声音特征 | 关键辨识点 |
|---------|---------|-----------|
| 金属小物（钥匙扣） | 高频"叮叮"、清脆、有余音 | 高频 + 余震 |
| 软胶公仔 | 低频"噗噗"、闷、无余音 | 闷 + 短促 |
| 毛绒挂件 | 几乎无声、轻微摩擦 | "听不太到"本身就是线索 |
| 多颗小件（豆袋） | "哗啦啦"连续碎响 | 数量感 |
| 重金属（摆件） | 沉闷"咚"、低频、慢 | 重量感 |
| 纸质卡片 | 轻微"沙沙"滑动 | 滑动质感 |

## 十、范围边界

| 做 | 不做 |
|---|------|
| 听声辨物 + 整组判定核心玩法 | 不做实时多人对战（首版） |
| 单机图鉴收集 | 不做好友交换系统（首版，后续可加） |
| IAA 广告变现 | 不做内购货币 / 付费抽卡（避免赌博合规风险） |
| 原创"丑萌" IP | **绝不使用 Labubu / Pop Mart / Molly 等商标**（侵权红线，只蹭品类热度） |
| 手配主题 + 周更 | 不做算法生成关卡（首版） |
| 断局不续玩 | 不做云存档 / 账号系统（首版本地存储即可） |
| 桌面/手机均可玩的 Web 原型 | 原型不做陀螺仪摇晃（用长按+晃动模拟），不做震动反馈 |

## 附：一句话总结

**玩家打开一套盲盒，靠耳朵把 6 个盒子听出来，押对 6 成就整组收走、听不准就打乱重来——每一刻在"摇—听—猜"，每一局在"赌自己的耳朵"，长期在"补全图鉴"。**

## 调优数值速查表（playtest 前全部为假设）

| 变量 | 基准值 | 最小 | 最大 | 调优笔记 |
|------|--------|------|------|---------|
| 单套盒子数 | 6 | 4 | 9 | 6 个约 3-5 分钟一局，适合碎片化 |
| 成功正确率线 | 60% | 50% | 70% | 6 个对 4 个；太高挫败，太低无成就 |
| 单盒免费摇晃次数 | 3 | 2 | 5 | 试：3 次够不够听清？ |
| 广告 +摇晃次数 | 2 | 1 | 3 | — |
| 隐藏款基础概率 | 5% | 3% | 8% | 翻倍后 10%，控制图鉴补全节奏 |
| 候选物品种类（首套） | 4 | 3 | 6 | 新手友好，越多越难 |
| 候选物品种类（高手套） | 8 | 6 | 12 | 技巧天花板 |
| 新手免广告局数 | 3 | 2 | 5 | 保护首次体验 |
| 插屏广告间隔 | 3 局 | 2 | 5 | 留存 vs 收入平衡 |
| 摇晃加速度阈值（手机） | 15 m/s² | 10 | 25 | 防误触，要手感 |
