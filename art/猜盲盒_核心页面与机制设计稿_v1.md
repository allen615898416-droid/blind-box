# 《猜盲盒（暂名）》核心页面与机制设计稿

> 版本 v1.0 · 2026-06-15 · GameDesigner
> 用途：**能直接进开发 / 喂给 AI 出图**的页面结构与机制定义。聚焦 4 个核心页面 + 主玩法机制。
> 品类：**收集 + 解谜**（全年龄向，非低龄）。游戏名「猜盲盒」为占位名，待定。
> 数值标 `[待测]`。本稿基于多轮机制定稿 + 用户实玩反馈，难度走 C 路线（主线轻松 + 挑战关烧脑）。

---

## 〇、一句话产品定义

> 在一面收集墙上看到一组想要的东西，点一个进去 —— 摇一摇盒子、凭声音/震动/晃动的感觉，从一组相似的盒子里猜出哪个是它，猜中就收入收集墙，集齐整套（含稀有隐藏款）。

**情绪**：好奇（里面是什么）+ 收集欲（想集齐）+ 小赌怡情（赌中的爽）。
**核心爽感压在**：拆盒仪式 + 收集满足 + 赌中惊喜。**辨别是调味，不是烧脑主菜。**

---

## 一、难度路线（C 路线，已拍板，统领全局）

| 层 | 难度 | 辨别差异 | 体验目标 | 占比 |
|----|------|---------|---------|------|
| **主线关** | 轻松 | 差别**明显**，一摇就能感觉出不同 | 不挫败、爽快、想继续 | 多数内容 |
| **挑战关** | 烧脑 | 差别**细微**，要反复对比品 | 满足硬核玩家、炫耀 | 少数内容 |

> 立项实测教训：差别做太小会让玩家"三四次过不去就弃游"。**主线必须保证轻松首胜**，差别宁可做大。难度通过"差异明显度 + 候选数量 + 探测次数"三个旋钮调节。

## 二、主玩法机制定义（探测—对比—押注—揭晓）

### 2.1 统一核心机制规格

| 字段 | 定义 |
|------|------|
| **目的** | 让玩家凭主观感知，从一组相似的盒子里辨别并赌中目标 |
| **玩家幻想** | 我有好直觉/好耳力，能赌赢 |
| **输入** | 进入一关 = 一组 N 个盒子（已知有哪些品类，未知谁在哪） |
| **核心动作** | ① 选中盒子 → ② 点「摇一摇」探测（限次）→ ③ 感受反馈 → ④ 组内切换对比 → ⑤ 给盒子押注 → ⑥ 确认整组 |
| **探测反馈（已定）** | **声音 + 震动 + 盒子动画幅度** 三者结合，凭直觉感受，**不用波形、不用关键词读数**（避免量化杀死探索） |
| **探测限次** | 每盒 `[待测: 3 次]`，用完可看激励广告 +2 次。**限次=赌性来源** |
| **揭晓** | 整组一次性揭晓，只告诉「押中几个」，不告诉哪几个（Mastermind 式） |
| **成功条件** | 押中目标藏品（主线：找出指定；挑战：全匹配） |
| **失败状态** | 没押中目标 → 本关重开（探测次数回满），**无保底、无惩罚、不清空已收集** |
| **设计红线（broken）** | ① 不摇就能猜对=差别太大（早教）② 摇了也分不出=差别太小（瞎蒙，会弃游）③ 把反馈做成关键词/数值读数=杀死探索 |

### 2.2 探测反馈三件套（核心呈现，开发须知）

> 探测一个盒子时，**同时**给三种感官反馈，让玩家"凭感觉"判断，而不是读数：

| 反馈通道 | 怎么表现 | 作用 |
|---------|---------|------|
| **声音** | 摇晃时播放该物品的音效（风声/撞击声/铃声等） | 戴耳机的主通道 |
| **震动** | 手机震动马达配合摇晃节奏（强/弱/连续/顿挫） | 触觉补充，增代入 |
| **动画幅度** | **盒子本身**摇晃/颤动的幅度与节奏（重物晃得沉、碎物抖得碎） | **不戴耳机时的主通道** |

**关键**：三者都是"现象级"反馈，要玩家**主观感受**，不在旁边放任何数值条/波形/关键词。差别的明显度由关卡难度（主线大/挑战小）控制。

> ⚠️ 开发注意：不戴耳机的玩家靠"动画幅度"也能玩——**动画幅度不是装饰，是核心线索通道之一**，必须做出可分辨的差异。

### 2.3 主线 vs 挑战关的差异

| | 主线关 | 挑战关 |
|--|--------|--------|
| 目标 | 找出指定的 1 个 | 一组全部匹配归位 |
| 差异明显度 | 大（一摇就感觉到） | 小（要反复对比品） |
| 候选数 | 少（3-4） | 多（5-6+） |
| 探测次数 | 宽松 | 偏紧 |

## 三、4 个核心页面结构定义

> 每页给：作用 / 区域结构（从上到下）/ 各区放什么 / 给 AI 的出图 prompt。竖屏手机。
> 美术风格：潮玩盲盒萌系（往泡泡玛特审美靠，全年龄、有质感、不幼齿）；糖果奶油色系；不复制 Labubu 具体形象/不用其商标。

### 页面 1 · 主页

**作用**：进入 App 的家，去各处的出发点，传达"想玩、有品质"。

**区域结构**：
1. 顶部条：游戏 Logo + 设置入口
2. 主视觉区（中部）：当期主推主题的明星藏品大图 + 「开始」主按钮
3. 进度区：总收集进度（如"已集 12/40"）+ 每日任务/奖励入口
4. 底部导航：主页 / 收集墙（两个 tab 起步）

**prompt**：
```
mobile game home screen UI concept, portrait, premium kawaii designer-toy aesthetic, pastel cream palette, 
top bar with game logo and a settings icon, 
center hero area showing a featured collectible character with a large rounded "start" button, 
a total collection progress bar and a daily reward entry below, 
bottom tab navigation, clean cozy high-quality casual game look, all-ages not childish
```

### 页面 2 · 收集墙（= 主题列表，二合一）

**作用**：核心驱动页。看见"拥有/缺什么"，选想要的进关。**缺口要勾人。**

**区域结构**：
1. 顶部：主题切换条（横向可滑的主题标签/卡片）
2. 主体：当前主题的**网格收集墙**
   - 已得格：点亮、显示藏品
   - 未得格：灰暗 + 问号「?」（缺口感）
   - 隐藏款格：金色 + 星标★（锁定时也可见，勾人）
3. 底部：该主题收集百分比 + 晒图/分享按钮

**prompt**：
```
mobile game collection wall screen UI concept, portrait, premium kawaii collectible aesthetic, 
top: a horizontal row of switchable theme tabs, 
main: a grid of collection slots — some filled with cute collectibles, some grayed with a question mark (missing), 
one special golden slot with a star (rare hidden edition, visible but locked), 
bottom: collection percentage and a share button, 
the empty slots should feel inviting to fill, pastel cream, cozy premium, all-ages
```

### 页面 3 · 核心游戏页（最重要，80% 体验）

**作用**：摇盒、感受反馈、组内对比、押注。**所有动作一页完成，不跳页。**

**区域结构（从上到下，固定骨架）**：
1. **目标条**：当前目标（"找出：大风"）+ 进度（"已押 2/4"）；若隐藏款触发，此处有醒目「⚡隐藏款」提示
2. **盒子组区**：一排/网格的盲盒（3-6 个），选中的高亮，已押注的显示押注标记
3. **反馈区**：选中盒子摇晃时，盒子在此**颤动/摇晃**（动画幅度=线索）；配声音、震动。**不放波形/数值/关键词**
4. **押注候选区**：候选项图标（带名字），点选押给当前盒
5. **操作区**：「摇一摇」按钮（显示剩余次数）+「确认整组」按钮

**prompt**：
```
mobile game core gameplay screen UI concept, portrait, premium kawaii aesthetic, 
top: objective hint "find: XX" with a small progress counter and an eye-catching hidden-edition banner, 
upper area: a row of cute mystery blind boxes, one highlighted as selected, the selected box visibly shaking/trembling, 
NO data readouts or waveform — the box's own shaking motion is the clue, 
lower area: a row of candidate answer icons to bet on the selected box, 
bottom: a "shake" button with remaining-count and a "confirm" button, 
clean uncluttered, boxes are the star, pastel cream, all-ages premium
```

### 页面 4 · 结果弹窗

**作用**：揭晓的高潮 + 收获满足。情绪爆发点。

**区域结构**（弹窗/覆盖层，非整页）：
1. 卡片顶部：结果状态（"找到了！" / "押中 3/4"）
2. 卡片中部：每个盒翻开显示真身 + 对/错标记；获得的藏品有"飞入收集墙"动效
3. 隐藏款解锁时：金色特别揭晓（闪光、星星，最显眼）
4. 卡片底部：两个按钮（"回收集墙" / "继续"）

**prompt**：
```
mobile game result popup UI concept, portrait, a celebratory reward card over a dimmed background, premium kawaii, 
top: happy result status like "you found it!" with a small "3/4 correct" tag, 
middle: boxes flipping open to reveal cute contents with check/cross marks, the won collectible flying into a collection, 
if hidden edition unlocked: a special golden reveal with sparkles, most eye-catching, 
bottom: two rounded buttons, joyful satisfying, pastel cream with soft gold celebration, all-ages
```
