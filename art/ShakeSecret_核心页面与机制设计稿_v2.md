# 《ShakeSecret（暂名：猜盲盒）》核心页面与机制设计稿

> 版本 v2.0 · 2026-06-15 · GameDesigner
> 用途：**能直接进开发 / 喂给 AI 出图**的页面结构与机制定义。聚焦 4 个核心页面 + 主玩法机制。
> 品类：**收集 + 解谜**（全年龄向，非低龄）。游戏名「ShakeSecret」为占位名，待定。
> 数值标 `[待测]`。本稿基于多轮机制定稿 + 用户实玩反馈 + Lovart 设计稿对齐。
> 核心模型：**真实开盲盒**（选一个盒子→Pick it→打开→直接看到内容，是啥是啥）。

---

## 〇、一句话产品定义

> 在一面收集墙上看到一组想要的东西，点一个进去 —— 摇一摇盒子、凭声音/震动/晃动的感觉，从几个相似的盒子里选一个打开，开出啥是啥，开出目标就点亮图鉴格，开出别的也收入囊中。

**情绪**：好奇（里面是什么）+ 收集欲（想集齐）+ 小赌怡情（选盒的赌）。
**核心爽感压在**：开盒仪式 + 收集满足 + 拆出惊喜。**辨别是调味，不是烧脑主菜。**

---

## 一、难度路线（C 路线，已拍板，统领全局）

| 层 | 难度 | 辨别差异 | 体验目标 | 占比 |
|----|------|---------|---------|------|
| **主线关** | 轻松 | 差别**明显**，一摇就能感觉出不同 | 不挫败、爽快、想继续 | 多数内容 |
| **挑战关** | 烧脑 | 差别**细微**，要反复对比品 | 满足硬核玩家、炫耀 | 少数内容 |

> 立项实测教训：差别做太小会让玩家"三四次过不去就弃游"。**主线必须保证轻松首胜**，差别宁可做大。

## 二、主玩法机制定义（探测—对比—选盒—开盒）

### 2.1 统一核心机制规格

| 字段 | 定义 |
|------|------|
| **目的** | 让玩家凭主观感知，从几个相似的盒子里选一个打开，体验真实开盲盒 |
| **玩家幻想** | 我有好直觉，能选中想要的那个 |
| **输入** | 进入一关 = 一组 N 个盒子（已知有哪些品类，未知谁在哪） |
| **核心动作** | ① 选中盒子 → ② 点「SHAKE」探测（总限次）→ ③ 感受反馈 → ④ 切换对比 → ⑤ 选一个盒子 → ⑥ 点「Pick it」→ ⑦ 打开盒子直接看内容 |
| **探测反馈（已定）** | **声音 + 震动 + 盒子动画幅度** 三者结合，凭直觉感受，**不用波形、不用关键词读数** |
| **探测限次** | 总探测次数 `[待测: 7 次]`（跨所有盒子共享），用完可看激励广告 +2 次。**限次=赌性来源** |
| **揭晓** | 打开选中的盒子，**直接展示内容（真实购买体验）**。是啥是啥 |
| **成功条件** | 开出目标角色 = 完成本关目标 |
| **失败状态** | 开出非目标 = 获得该角色并点亮对应图鉴格（如未收集），但未完成本关目标，可重开 |
| **设计红线** | ① 不摇就能猜对=差别太大（早教）② 摇了也分不出=差别太小（瞎蒙，会弃游）③ 把反馈做成关键词/数值读数=杀死探索 |

### 2.2 探测反馈三件套

> 探测一个盒子时，**同时**给三种感官反馈，让玩家"凭感觉"判断。
> **完整技术规格见《摇盒探测反馈三件套_设计规格_v1.md》**

| 反馈通道 | 怎么表现 | 作用 |
|---------|---------|------|
| **声音** | 摇晃时播放该物品的音效（基于真实录音+7维基因参数实时变换） | 戴耳机的主通道 |
| **震动** | 手机震动马达配合摇晃节奏（强/弱/连续/顿挫），从音效基因推导 | 触觉补充，增代入 |
| **动画幅度** | **盒子本身**摇晃/颤动的幅度与节奏（重物晃得沉、碎物抖得碎），从音效基因推导 | **不戴耳机时的主通道** |

### 2.3 主线 vs 挑战关的差异

| | 主线关 | 挑战关 |
|--|--------|--------|
| 目标 | 找出指定的 1 个 | 从更多盒子里找出目标 |
| 盒子数 | 3 | 5+ |
| 差异明显度 | 大（一摇就感觉到） | 小（要反复对比品） |
| 候选数 | 少（4） | 多（6+） |
| 探测次数 | 宽松（7） | 偏紧（5） |

## 三、4 个核心页面结构定义

> 对齐 Lovart v3 设计稿（已通过审查）。竖屏手机，画布 750×1334px @2x。
> 美术风格：潮玩盲盒萌系 IP，往泡泡玛特审美靠（全年龄、有质感、不幼齿）；糖果奶油色系。
> 当前系列：COZY FRIENDS（Pip 小黄鸡系列，中性暖黄风格）。

### 页面 1 · 主页

**作用**：进入 App 的家，传达"想玩、有品质"。

**区域结构**：
1. 顶部条：游戏 Logo + 设置入口
2. 主视觉区（中部）：当期主推主题的明星藏品大图 + **START** 主按钮（粉色胶囊）
3. 底部：**Collection Guide** 入口（跳转收集墙）

| 页面 | 进入条件 | 离开条件 | 页面上有什么 |
|------|---------|---------|-------------|
| **主页** | 启动后 | 点 Collection Guide / START | Logo、主推主题、START 按钮、Collection Guide 入口 |

### 页面 2 · 收集墙

**作用**：核心驱动页。看见"拥有/缺什么"，选想要的进关。**缺口要勾人。**

**区域结构**：
1. ① **系列 Tab 栏**：5 个可滑动 Tab（COZY FRIENDS / SWEET DREAMS / ...），当前高亮+下划线
2. ② **系列名行 + 👁 预览**：`✦ COZY FRIENDS ✦` + 右侧👁预览图标（点击弹全家福浮窗）
3. ③ **3×3 网格**：当前主题的收集墙
   - 已得格：点亮、显示藏品（Pip / Mochi / Biscuit / ...）
   - 未得格：灰暗 + 问号「?」
   - 隐藏款格：金色 + 星标★（锁定时也可见，勾人）
4. ④ **进度条 + SHARE**：收集百分比（如 89%）+ 分享按钮

| 页面 | 进入条件 | 离开条件 | 页面上有什么 |
|------|---------|---------|-------------|
| **收集墙** | 主页/结算 | 点某藏品进关 / ✕返回 | 系列Tab、系列名+👁、3×3网格、进度条、SHARE |

### 页面 3 · 核心游戏页（最重要，80% 体验）

**作用**：摇盒、感受反馈、选盒开盒。**所有动作一页完成，不跳页。**

**区域结构（从上到下，固定骨架）**：
1. **find 目标区**：`find: [目标角色轮廓?] (0/1)`；若隐藏款触发，此处有醒目「⚡隐藏款」提示
2. **3 盒 carousel**：3 个盒子，中间大两侧小+半透明，**外观完全相同**（同色同图案同"?"），左右滑动或点击切换
3. **SHAKE 按钮 + ×7**：点击触发当前盒探测（总次数 ×7）
4. **候选角色行**：候选项（轮廓/emoji + 名字），点选押给选中盒子
5. **Pick it 按钮**：底部全宽主按钮，提交选择——打开选中的盒子

| 页面 | 进入条件 | 离开条件 | 页面上有什么 |
|------|---------|---------|-------------|
| **核心游戏页** | 收集墙点击某藏品 | Pick it / 退出 | find区+⚡隐藏款、3盒carousel、SHAKE×7、候选行、Pick it |

> ⚠️ 三个盒子**必须外观完全相同**——同颜色、同图案、同"?"，只有大小不同（中间大、两侧小+半透明）。绝不在任何盒子上标注"HIDDEN EDITION"或任何稀有度暗示。

### 页面 4 · 结果页

**作用**：开盒的高潮 + 收获满足。情绪爆发点。

**区域结构（全屏覆盖层，深色背景）**：
1. **一个盒子**：选中的盒子打开，角色跳出（生动姿态，非贴纸感），光芒+碎片粒子
2. **结果状态**：`Got [角色名]!` / `NEW!`（首次获得）/ `Already owned`（已有）
3. **隐藏款解锁时**：金色特别揭晓（闪光、星星，最显眼）
4. **主按钮**：**Continue**（文案按场景变："Next Round" / "Back to Wall" / "Keep Going"）
5. **Mini 收集预览**：底部一排小头像，展示收集进度

| 页面 | 进入条件 | 离开条件 | 页面上有什么 |
|------|---------|---------|-------------|
| **结果页** | Pick it | Continue | 盒子打开、Got/NEW/Already owned、隐藏款金色揭晓、Continue、Mini预览 |

```
                  ↑__________________________________________|
                         (回收集墙 / 继续下一关)
```
